Чем GPU-параллелизм в архитектуре Transformer отличается от параллелизма в прежних NLP-алгоритмах?

## 1. Что такое параллельные вычисления на GPU?

Параллельные вычисления на GPU — это технология, которая использует графический процессор (GPU) для массовой параллельной обработки данных. По сравнению с традиционным центральным процессором (CPU), GPU имеет гораздо больше ядер и способен одновременно обрабатывать тысячи потоков. Благодаря этому GPU отлично подходит для задач с высокой степенью параллелизма. В глубоком обучении параллельные вычисления на GPU широко применяются при обучении нейронных сетей и ускоряют процесс тренировки моделей.

До 2017 года исследователи в области обработки естественного языка (NLP) обычно обучали модели с нуля, и уже сама возможность использовать GPU для обучения считалась продвинутой конфигурацией. Хотя у некоторых исследователей было несколько GPU, они часто не тратили силы на реализацию параллельных вычислений: тогдашние модели были сравнительно небольшими, обучение проходило относительно быстро, поэтому дополнительные затраты на параллелизм казались не слишком оправданными.

Однако со временем большие языковые модели (LLM) начали выходить на первый план. У таких моделей огромное число параметров, а обучение стало долгим и сложным процессом. Чтобы эффективно справляться с этими вызовами, обучение на нескольких GPU стало обычной практикой, а технологии параллельных вычислений — необходимостью. Особенно с учетом уникальной архитектуры моделей Transformer традиционные методы параллельных вычислений потребовали соответствующих изменений и оптимизаций, чтобы подходить требованиям этого нового типа моделей.

## 2. Типы параллельных вычислений

В глубоком обучении параллельные вычисления на GPU в основном делятся на два типа: параллелизм данных и параллелизм модели.

![alt text](../../../05-第五章-显卡与并行/assest/Transformer架构的GPU并行和之前的NLP算法有什么不同/0.png)

- ***Distributed Data Parallel (DDP)*** Параллелизм данных означает, что модель не слишком велика и целиком помещается в память одной GPU. Поэтому, чтобы ускорить обучение, данные делят на несколько частей и загружают на разные GPU. Параметры модели на каждой GPU одинаковые; данные каждого batch вычисляются на разных GPU, после чего градиенты агрегируются для обновления параметров.

![alt text](../../../05-第五章-显卡与并行/assest/Transformer架构的GPU并行和之前的NLP算法有什么不同/1.png)

- Параллелизм модели означает, что модель слишком велика и не может целиком поместиться в память одной GPU. Поэтому модель делят на несколько частей и загружают на разные GPU. Параметры модели на каждой GPU различаются; данные каждого batch вычисляются на разных GPU, после чего градиенты агрегируются для обновления параметров.



## 3. Что нужно загружать в видеопамять GPU?

В больших моделях в основном нужно загружать следующие части:

- Параметры модели
- Состояние оптимизатора модели: включает состояния прямого и обратного прохода
- Градиенты модели
- Другие промежуточные состояния модели

Важно отметить: на рисунке ниже все показано на примере FP32. В реальном обучении мы можем использовать смешанную точность: например, FP16 для параметров модели, FP32 для градиентов и так далее. В некоторых статьях и бенчмарках явно не указано, используется ли смешанная точность, поэтому при расчете доли видеопамяти могут возникать расхождения.

## 4. FSDP

***Fully Sharded Data Parallel (FSDP)***

FSDP (Fully Sharded Data Parallel) — это продвинутая технология параллелизма данных, которая позволяет эффективно обучать большие модели на нескольких GPU. FSDP впервые был предложен в FairScale-FSDP, а затем интегрирован в PyTorch 1.11. Он похож на уровень ZERO-3 алгоритма ZERO во фреймворке Microsoft Deepspeed.

В традиционном параллелизме данных (DDP) каждая GPU хранит все параметры модели, градиенты и состояние оптимизатора. Затем датасет делится на несколько шардов (shards), и каждая GPU отвечает за обучение на своей части. После вычисления градиентов они объединяются с помощью операции all-reduce.

Ключевая идея FSDP состоит в шардировании градиентов модели, состояния оптимизатора и параметров, благодаря чему каждая GPU хранит только часть информации о параметрах. Главное здесь — разложить операцию all-reduce из DDP на операции reduce-scatter и all-gather. Во время прямого прохода в FSDP каждая GPU должна с помощью all-gather получить полный набор параметров, затем выполнить вычисления и после этого отбросить параметры из чужих шардов. Во время обратного прохода также нужно с помощью all-gather получить полный набор параметров и вычислить градиенты для локального batch. Наконец, с помощью reduce-scatter на устройствах выполняется усреднение и шардирование градиентов, и каждое устройство обновляет только соответствующий ему шард параметров.

![alt text](../../../05-第五章-显卡与并行/assest/Transformer架构的GPU并行和之前的NLP算法有什么不同/3.png)

У FSDP есть четыре режима:

- stage0: вырождается в DDP
- stage1: шардируется только состояние оптимизатора модели
- stage2: шардируются градиенты модели и состояние оптимизатора
- stage3: шардируются градиенты модели, состояние оптимизатора и параметры

## 5. Offload

Offload — это технология переноса части параметров модели из памяти GPU в память CPU. При обучении больших моделей из-за огромного числа параметров иногда возникает нехватка памяти GPU. Чтобы решить эту проблему, можно использовать Offload и перенести часть параметров из памяти GPU в память CPU, тем самым освободив память GPU и позволив модели продолжать обучение.

Технология Offload в FSDP может переносить часть параметров модели из памяти GPU в память CPU.

## 6. Итоги

Одно из больших преимуществ Transformer заключается в том, что он хорошо использует возможности параллельных вычислений на GPU. При обучении больших моделей нужно учитывать число параметров модели и ограничения памяти GPU; в таких случаях требуются технологии вроде FSDP и Offload, чтобы оптимизировать процесс обучения и повысить его эффективность.

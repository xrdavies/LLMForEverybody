Фреймворки инференса больших моделей (часть 1): обзор

## 1. Почему нужны фреймворки инференса

Помимо распределенного инференса и поддержки квантизации, главная польза фреймворков инференса больших моделей заключается в ускорении инференса. Основная цель ускорения инференса - повысить его эффективность, снизить требования к вычислениям и памяти, удовлетворить требованиям реального времени и уменьшить стоимость развертывания:

**Вычисления и память**: большие модели обычно требуют значительных вычислительных ресурсов и памяти для обработки сложных задач. В сценариях с ограниченными ресурсами, например на мобильных устройствах или в средах edge computing, это приводит к низкой эффективности инференса. Чтобы решить эту проблему, необходимо повышать эффективность инференса с помощью методов оптимизации и снижать требования к вычислениям и памяти;

**Реальное время**: во многих прикладных сценариях, таких как голосовые ассистенты и перевод в реальном времени, пользователи ожидают мгновенной обратной связи. Скорость инференса большой модели напрямую влияет на пользовательский опыт. Поэтому ускорение инференса позволяет уменьшить latency и обеспечить более плавное взаимодействие;

**Стоимость развертывания**: развертывание больших моделей требует дорогой аппаратной поддержки, например высокопроизводительных GPU. Благодаря ускорению инференса большие модели можно развертывать на более дешевом оборудовании, снижая стоимость развертывания и делая применение больших моделей более широким;

**Оптимизация производительности системы**: при реальном развертывании производительность инференса больших моделей зависит от факторов системного уровня, таких как пропускная способность памяти и утилизация вычислительных блоков. Оптимизация на уровне системы позволяет повысить скорость и эффективность инференса больших моделей.

## 2. Технологии ускорения инференса

Чтобы повысить эффективность инференса больших моделей, обычно применяют следующие технологии:

1. **Повышение вычислительной эффективности**: MQA (Multi Query Attention) за счет параллельного вычисления распределения внимания для нескольких векторов query может заметно повысить вычислительную эффективность. В MQA все heads совместно используют одну и ту же матрицу Key и Value, а каждый head отдельно хранит только параметры Query, что значительно уменьшает количество параметров матриц Key и Value и ускоряет генерацию token в decoder;

2. **Сокращение объема вычислений**: GQA (Group Query Attention) группирует несколько векторов query для вычисления распределения внимания. Каждая группа совместно использует общую проекцию key (K) и value (V), что снижает накладные расходы памяти на хранение key и value для каждого head, особенно при большом контекстном окне или размере batch;

![alt text](../../../02-第二章-部署与推理/assest/大模型推理框架（一）综述/0.PNG)

3. **Снижение сложности**: Flash Attention повышает вычислительную эффективность за счет уменьшения сложности вычислений внимания. С помощью блочного вычисления (Tiling) и повторного вычисления (Recomputation) он избегает явного построения и хранения больших матриц внимания, тем самым существенно повышая скорость вычислений и эффективность использования памяти;

![alt text](../../../02-第二章-部署与推理/assest/大模型推理框架（一）综述/1.png)

4. **Сокращение повторных вычислений**: KV Cache (кэш ключей и значений) повышает эффективность инференса за счет кэширования пар key-value в вычислениях внимания и уменьшения объема повторных вычислений. В авторегрессионных моделях KV Cache использует информацию Key и Value из повторяющегося инференса, избегая повторного вычисления на каждом шаге инференса и тем самым уменьшая объем вычислений параметров;

![alt text](../../../02-第二章-部署与推理/assest/大模型推理框架（一）综述/2.png)

5. **Повышение throughput**: Continuous Batching (непрерывная батч-обработка) повышает вычислительную эффективность и throughput за счет непрерывной обработки нескольких входных примеров. Этот метод особенно подходит для развертывания LLM, поскольку позволяет эффективнее использовать память GPU, сокращать время простоя и тем самым повышать общую скорость инференса.

## 3. Распространенные фреймворки инференса

1. `vLLM`[1](#refer-anchor-1) - это фреймворк инференса на основе PagedAttention. Он реализует эффективный, быстрый и недорогой LLM-сервис за счет постраничной обработки вычислений внимания. Во время инференса vLLM делит вычисления внимания на несколько страниц, каждая из которых вычисляет только часть распределения внимания, тем самым снижая объем вычислений и требования к памяти и повышая эффективность инференса;

2. `Text Generation Inference（TGI）`[2](#refer-anchor-2) - это фреймворк для развертывания и обслуживания больших языковых моделей (LLMs), разработанный Hugging Face. Это production-grade инструментарий, специально спроектированный для запуска больших языковых моделей в виде сервиса на локальных машинах. TGI написан на Rust и Python и предоставляет endpoint для вызова модели, делая задачи генерации текста более эффективными и гибкими;

3. `TensorRT-LLM` [3](#refer-anchor-3) - это open-source библиотека NVIDIA для оптимизации производительности инференса больших языковых моделей (LLMs) на NVIDIA GPU. Благодаря набору продвинутых методов оптимизации, таких как квантизация, слияние kernel, динамическая батч-обработка и поддержка нескольких GPU, она значительно повышает скорость инференса LLMs; по сравнению с традиционными CPU-based подходами скорость инференса может вырасти до 8 раз;

4. `Ollama`[4](#refer-anchor-4) - это open-source фреймворк, предназначенный для упрощения развертывания и запуска больших языковых моделей (LLM) в локальной среде. Он достигает этого за счет упаковки весов модели, конфигурации и датасетов в единый пакет под названием Modelfile, что делает скачивание, установку и использование LLM гораздо удобнее даже для пользователей без глубоких технических знаний;

5. `DeepSpeed`[5](#refer-anchor-5) - это библиотека оптимизации deep learning от Microsoft. Она поддерживает не только обучение крупномасштабных моделей, но и ускорение инференса. DeepSpeed-Inference является частью DeepSpeed и сфокусирован на повышении производительности инференса больших моделей, включая latency и throughput.

Фреймворки инференса больших моделей (часть 2): vLLM

`vLLM`[1](#refer-anchor-1) - это фреймворк инференса на основе PagedAttention. Он реализует эффективный, быстрый и недорогой LLM-сервис за счет постраничной обработки вычислений внимания. Во время инференса vLLM делит вычисления внимания на несколько страниц, каждая из которых вычисляет только часть распределения внимания, тем самым снижая объем вычислений и требования к памяти и повышая эффективность инференса.

## 1. PagedAttention

Узкое место производительности LLM-сервисов находится в памяти (видеопамяти). В процессе авторегрессионного декодирования autoregressive decoding все входные tokens LLM создают свои attention key and value tensors; эти тензоры сохраняются в памяти GPU для генерации следующих tokens. Такие кэшированные key и value tensors обычно называют KV Cache (KV cache). У KV Cache есть следующие особенности:

- Большой объем: в LLaMA-13B одна последовательность может занимать до 1.7GB.
- Динамичность: размер зависит от длины последовательности, а длина последовательности сильно варьируется и непредсказуема. Поэтому эффективное управление KV Cache является серьезной задачей. Мы обнаружили, что существующие системы из-за фрагментации и чрезмерного резервирования теряют 60% - 80% памяти.

Чтобы решить эту проблему, vLLM представил PagedAttention - алгоритм внимания, вдохновленный классическими идеями виртуальной памяти и paging в операционных системах. В отличие от традиционных алгоритмов внимания, PagedAttention позволяет хранить непрерывные key и value в несмежных областях памяти. Конкретнее, PagedAttention делит KV Cache каждой последовательности на blocks, и каждый block содержит key и value для фиксированного числа token. Во время вычисления внимания kernel PagedAttention эффективно определяет и извлекает эти blocks.

![PagedAttention: KV Cache разделяется на blocks. Blocks не обязаны быть непрерывными в пространстве памяти.](../../../02-第二章-部署与推理/assest/大模型推理框架（二）vLLM/0.gif)

Поскольку blocks не обязаны быть непрерывными в памяти, key и value можно управлять более гибко, как в виртуальной памяти операционной системы: blocks можно рассматривать как pages, tokens - как bytes, а sequences - как processes. Непрерывные ***логические blocks*** ***logical blocks*** последовательности отображаются через **block table** **таблицу блоков** на несмежные **физические blocks** ***physical blocks***. Физические blocks выделяются по мере необходимости при генерации новых tokens.

![Пример процесса генерации для запроса, который сэмплирует несколько выходов](../../../02-第二章-部署与推理/assest/大模型推理框架（二）vLLM/1.gif)

В PagedAttention потери памяти возникают только в последнем block последовательности. На практике это дает почти оптимальное использование памяти: доля потерь ниже 4%. Повышение эффективности памяти оказалось очень полезным: оно позволяет системе объединять в batch больше последовательностей, повышает утилизацию GPU и тем самым значительно увеличивает throughput, как показано в результатах производительности на рисунке выше.

У PagedAttention есть еще одно ключевое преимущество: эффективное совместное использование памяти. Например, при parallel sampling один и тот же prompt генерирует несколько выходных последовательностей. В таком случае вычисления и память для этого prompt могут совместно использоваться между выходными последовательностями.

![Пример parallel sampling](../../../02-第二章-部署与推理/assest/大模型推理框架（二）vLLM/2.gif)

PagedAttention естественным образом реализует совместное использование памяти через block table. Подобно тому, как процессы совместно используют физические страницы, разные последовательности в PagedAttention могут совместно использовать blocks, отображая свои logical blocks на один и тот же physical block. Чтобы обеспечить безопасное совместное использование, PagedAttention отслеживает счетчик ссылок physical blocks и реализует механизм copy-on-write.

![Пример процесса генерации для запроса, который сэмплирует несколько выходов](../../../02-第二章-部署与推理/assest/大模型推理框架（二）vLLM/3.gif)

Функция совместного использования памяти в PagedAttention существенно снижает накладные расходы памяти для сложных алгоритмов сэмплирования, таких как parallel sampling и beam search: использование памяти может уменьшаться до 55%. Это может превращаться в прирост throughput до 2.2 раза. Благодаря этому такие методы сэмплирования становятся практичными для LLM-сервисов.

PagedAttention - ключевая технология, лежащая в основе vLLM. vLLM - это наш движок инференса и обслуживания LLM, который поддерживает множество моделей, обладает высокой производительностью и удобным интерфейсом.

## 2. Другие возможности vLLM

Помимо **алгоритма PagedAttention**, vLLM благодаря ряду оптимизаций позволяет эффективно развертывать большие языковые модели в production-средах. Ниже приведены некоторые ключевые возможности vLLM:

1. **Алгоритм PagedAttention**: vLLM использует новый алгоритм внимания PagedAttention, который эффективно управляет памятью key и value в механизме внимания, снижает потребление видеопамяти и повышает throughput модели.

2. **Поддержка нескольких GPU**: vLLM поддерживает multi-GPU системы, может эффективно распределять workload, уменьшать bottleneck и увеличивать общий throughput системы.

3. **Непрерывная батч-обработка**: vLLM поддерживает continuous batching и позволяет динамически распределять задачи. Это особенно полезно в средах с колеблющимся workload: можно сокращать время простоя и повышать эффективность управления ресурсами.

4. **Speculative decoding**: vLLM оптимизирует latency Chatbots и генераторов текста в реальном времени за счет предварительной генерации и проверки потенциальных будущих tokens, сокращая время инференса LLM.

5. **Оптимизированное использование памяти**: embedding layer в vLLM глубоко оптимизирован с точки зрения эффективности памяти, что обеспечивает сбалансированное и эффективное использование GPU memory в LLM и решает проблемы ресурсов памяти большинства LLM (например, ChatGPT) без потери производительности.

6. **LLM-адаптеры**: vLLM поддерживает интеграцию LLM-адаптеров, позволяя разработчикам донастраивать и кастомизировать LLM без переобучения всей системы, что обеспечивает гибкость.

7. **Бесшовная интеграция с моделями HuggingFace**: пользователи могут напрямую использовать vLLM на платформе HuggingFace для инференса и обслуживания моделей без дополнительных разработческих работ.

8. **Поддержка распределенного инференса**: vLLM поддерживает tensor parallelism и pipeline parallelism, предоставляя пользователям гибкое и эффективное решение.

9. **Совместимость с сервисом OpenAI API**: vLLM обеспечивает совместимость с интерфейсом OpenAI API, благодаря чему пользователям проще интегрировать vLLM в существующие системы.

10. **Поддержка технологий квантизации**: vLLM поддерживает int8-квантизацию, уменьшает размер модели, повышает скорость инференса и помогает развертывать большие языковые модели на устройствах с ограниченными ресурсами.

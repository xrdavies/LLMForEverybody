## Основной текст:

LLM обещают радикально изменить то, как мы используем AI во всех отраслях. Однако построить реальный сервис для модели довольно сложно: он может оказаться неожиданно медленным даже на дорогом оборудовании. Сегодня, 20 июня 2023 года, прим. переводчика, мы рады представить vLLM - open-source библиотеку для быстрого LLM-инференса и обслуживания. vLLM использует PagedAttention, наш новый алгоритм внимания, который эффективно управляет ключами (`keys`) и значениями (`values`) внимания. vLLM с PagedAttention переопределяет SOTA-уровень LLM-сервисов: его throughput до 24 раз выше, чем у HuggingFace Transformers, и для этого не нужны изменения архитектуры модели.

vLLM разработан в UC Berkeley и последние два месяца использовался внутри [Chatbot Arena and Vicuna Demo](https://lmarena.ai/). Это ключевая технология, которая делает LLM-сервисы экономичными и доступными даже для небольших исследовательских команд с ограниченными вычислительными ресурсами, таких как LMSYS. Теперь vLLM можно попробовать всего одной командой из нашего [GitHub-репозитория](https://github.com/vllm-project/vllm).

## Производительность выше SOTA

Мы сравнили throughput vLLM с самой популярной LLM-библиотекой [HuggingFace Transformers (HF)](https://huggingface.co/docs/transformers/main_classes/text_generation) и предыдущей SOTA-системой [HuggingFace text-generation-inference (TGI)](https://github.com/huggingface/text-generation-inference). Оценка проводилась в двух конфигурациях: LLaMA-7B на NVIDIA A10G GPU и LLaMA-13B на NVIDIA A100 GPU (40GB). Длины входа/выхода запросов мы брали из датасета ShareGPT. В наших экспериментах throughput vLLM был в 24 раза выше, чем у HF, и в 3.5 раза выше, чем у TGI.

## Секрет: PagedAttention

В vLLM мы обнаружили, что узкое место производительности LLM-сервиса находится в памяти, то есть в видеопамяти. В процессе autoregressive decoding все input tokens LLM создают свои тензоры ключей и значений внимания (`attention key and value tensors`), которые сохраняются в GPU-памяти для генерации следующих tokens. Эти кэшированные тензоры ключей и значений обычно называют KV Cache (`KV cache`). У KV Cache есть следующие особенности:

- Большой размер: в LLaMA-13B одна последовательность занимает до 1.7GB.
- Динамичность: размер зависит от длины последовательности, а длина последовательности сильно меняется и непредсказуема. Поэтому эффективное управление KV Cache - серьезная задача. Мы обнаружили, что существующие системы из-за фрагментации и чрезмерного резервирования тратят впустую 60%-80% памяти.

Чтобы решить эту проблему, мы представили PagedAttention - алгоритм внимания, вдохновленный классическими идеями виртуальной памяти и paging в операционных системах. В отличие от традиционных алгоритмов внимания, PagedAttention позволяет хранить последовательные ключи и значения в несмежных областях памяти. Конкретнее, PagedAttention делит KV Cache каждой последовательности на блоки, и каждый блок содержит ключи и значения фиксированного числа token. Во время вычисления внимания ядро PagedAttention эффективно определяет и получает эти блоки.

![alt text](<../../../02-第二章-部署与推理/assest/vLLM 使用PagedAttention轻松、快速且廉价地提供LLM服务（中文版翻译）/2.gif>)

PagedAttention: KV Cache разделен на блоки. Блоки не обязаны быть смежными в пространстве памяти.

Поскольку blocks не обязаны быть смежными в памяти, мы можем управлять keys & values более гибко, как в виртуальной памяти операционной системы: blocks можно рассматривать как pages, tokens - как bytes, а sequences - как processes. Последовательные ***логические блоки*** (`logical blocks`) последовательности через **таблицу блоков** (`block table`) отображаются на несмежные ***физические блоки*** (`physical blocks`). Физические блоки выделяются по требованию при генерации новых tokens.

![alt text](<../../../02-第二章-部署与推理/assest/vLLM 使用PagedAttention轻松、快速且廉价地提供LLM服务（中文版翻译）/3.gif>)

Пример процесса генерации запроса с PagedAttention.

В PagedAttention потеря памяти возникает только в последнем блоке последовательности. На практике это дает почти оптимальное использование памяти: доля потерь ниже 4%. Повышение эффективности памяти оказалось очень полезным: система может объединять в batch больше последовательностей, повышать утилизацию GPU и тем самым значительно увеличивать throughput, как показано в результатах производительности выше.

У PagedAttention есть еще одно ключевое преимущество: эффективное разделение памяти. Например, при parallel sampling один и тот же prompt генерирует несколько выходных последовательностей. В таком случае вычисления и память этого prompt могут совместно использоваться выходными последовательностями.

![alt text](<../../../02-第二章-部署与推理/assest/vLLM 使用PagedAttention轻松、快速且廉价地提供LLM服务（中文版翻译）/4.gif>)

Пример parallel sampling.

PagedAttention естественным образом реализует разделение памяти через свою таблицу блоков. Подобно тому как процессы совместно используют физические страницы, разные последовательности в PagedAttention могут совместно использовать блоки, отображая свои логические блоки на один и тот же физический блок. Чтобы обеспечить безопасное совместное использование, PagedAttention отслеживает счетчик ссылок физических блоков и реализует механизм copy-on-write.

![alt text](<../../../02-第二章-部署与推理/assest/vLLM 使用PagedAttention轻松、快速且廉价地提供LLM服务（中文版翻译）/5.gif>)

Пример процесса генерации запроса при sampling нескольких выходов.

Возможность разделения памяти в PagedAttention значительно снижает накладные расходы памяти у сложных алгоритмов sampling, например `parallel sampling` и `beam search`, и может уменьшить использование памяти до 55%. Это может преобразоваться в прирост throughput до 2.2 раза. Благодаря этому такие методы sampling становятся практичными в LLM-сервисах.

PagedAttention - ключевая технология за vLLM, нашим движком LLM-инференса и обслуживания. vLLM поддерживает множество моделей, обладает высокой производительностью и удобным интерфейсом. Больше технических деталей о vLLM и PagedAttention смотрите в нашем [GitHub-репозитории](https://github.com/vllm-project/vllm) и следите за нашей статьей.

## Скрытый герой LMSYS Vicuna and Chatbot Arena

В апреле этого года, апрель 2023 года, прим. переводчика, LMSYS разработала и открыла популярную чат-бот модель Vicuna. С тех пор Vicuna обслужила миллионы пользователей в [Chatbot Arena](https://lmarena.ai/). Изначально LMSYS FastChat использовал [serving backend](https://github.com/lm-sys/FastChat/blob/main/fastchat/serve/model_worker.py) на базе HF Transformers для обслуживания chat demo. По мере роста популярности demo пиковый трафик увеличился в несколько раз, и HF backend стал серьезным узким местом. Команды LMSYS и vLLM совместно и быстро разработали интеграцию FastChat-vLLM, чтобы использовать vLLM как [новый backend](https://github.com/lm-sys/FastChat/blob/main/fastchat/serve/vllm_worker.py) для поддержки растущего спроса, вплоть до 5-кратного трафика. В ранних [внутренних микробенчмарках](https://github.com/lm-sys/FastChat/blob/main/fastchat/serve/test_throughput.py) LMSYS serving backend на vLLM достигал **throughput в 30 раз выше, чем у исходного HF backend**.

С середины апреля 2023 года самые популярные модели, включая Vicuna, Koala и LLaMA, успешно обслуживались через интеграцию FastChat-vLLM. Используя FastChat как frontend мультимодельного chat-сервиса, а vLLM как backend инференса, LMSYS смогла с ограниченным числом университетских спонсорских GPU обслужить миллионы пользователей Vicuna с высоким throughput и низкой latency. LMSYS расширяет применение vLLM на более широкий набор моделей, включая Databricks Dolly, OpenAsssiant от LAION и stableLM от Stability AI. Поддержка большего числа [моделей](https://docs.vllm.ai/en/latest/models/supported_models.html) находится в разработке и скоро появится.

![alt text](<../../../02-第二章-部署与推理/assest/vLLM 使用PagedAttention轻松、快速且廉价地提供LLM服务（中文版翻译）/6.png>)

Запросы, обработанные интеграцией FastChat-vLLM в Chatbot Arena в период с апреля по май. Фактически больше половины запросов Chatbot Arena использовали vLLM как backend инференса.

Использование vLLM также значительно снизило операционные затраты. Благодаря vLLM LMSYS смогла сократить количество GPU, необходимых для обработки указанного трафика, на 50%. vLLM в среднем обрабатывает 30K запросов в день, а пик достигает 60K, что хорошо демонстрирует устойчивость vLLM.

## Начало работы с vLLM

Установите vLLM следующей командой, больше информации смотрите в нашем [руководстве по установке](https://docs.vllm.ai/en/latest/getting_started/installation.html):

```shell
$ pip install vllm
```

vLLM можно использовать и для offline-инференса, и для online serving. Чтобы использовать vLLM для offline-инференса, импортируйте vLLM и используйте класс LLM в Python-скрипте:

```python
from vllm import LLM

prompts = ["Hello, my name is", "The capital of France is"]  # Sample prompts.
llm = LLM(model="lmsys/vicuna-7b-v1.3")  # Create an LLM.
outputs = llm.generate(prompts)  # Generate texts from the prompts.
```

Чтобы использовать vLLM для online serving, можно запустить сервер, совместимый с OpenAI API, следующим способом:

```shell
$ python -m vllm.entrypoints.openai.api_server --model lmsys/vicuna-7b-v1.3
```

Вы можете запрашивать сервер в том же формате, что и OpenAI API:

```shell
$ curl http://localhost:8000/v1/completions \
    -H "Content-Type: application/json" \
    -d '{
        "model": "lmsys/vicuna-7b-v1.3",
        "prompt": "San Francisco is a",
        "max_tokens": 7,
        "temperature": 0
    }'

```

Больше способов использования vLLM смотрите в [quickstart guide](https://docs.vllm.ai/en/latest/getting_started/quickstart.html).

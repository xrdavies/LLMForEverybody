`Text Generation Inference (TGI)`[1](#refer-anchor-1) - это фреймворк, разработанный Hugging Face для развертывания и обслуживания больших языковых моделей (`LLMs`). Это production-ready набор инструментов, специально созданный для запуска больших языковых моделей как сервиса на локальных машинах. TGI написан на Rust и Python и предоставляет endpoint для вызова модели, делая задачи генерации текста более эффективными и гибкими.

## 1. Технологии ускорения инференса

1. Tensor Parallelism

Tensor parallelism использует свойство матричного умножения, позволяющее выполнять вычисления параллельно: параметры модели делятся на несколько частей, каждая часть вычисляется на отдельном устройстве, а затем результаты объединяются. Ниже отдельно рассмотрим реализацию tensor parallelism для FFN и Self-Attention.

Основные строительные блоки `MLP` - это полностью связанные `nn.Linear`, за которыми следует нелинейная активация GeLU.

Следуя обозначениям из статьи Megatron, его часть со скалярным произведением можно записать как `Y = GeLU(XA)`, где `X` и `Y` - входной и выходной векторы, а `A` - матрица весов.

Если посмотреть на вычисления в матричной форме, легко увидеть, как матричное умножение можно разделить между несколькими GPU:

![alt text](<../../../02-第二章-部署与推理/assest/大模型推理框架（三）Text generation inference (TGI)/1.png>)

Если разделить матрицу весов `A` по столбцам между `N` GPU и параллельно выполнить матричные умножения `XA_1` до `XA_n`, мы получим `N` выходных векторов `Y_1`, `Y_2`, ..., `Y_n`, которые можно независимо передать в GeLU:

![alt text](<../../../02-第二章-部署与推理/assest/大模型推理框架（三）Text generation inference (TGI)/2.png>)

Используя этот принцип, можно обновлять MLP любой глубины без синхронизации между GPU вплоть до самого конца, когда потребуется восстановить выходной вектор.

Авторы статьи Megatron-LM приводят для этого полезный пример:

![alt text](<../../../02-第二章-部署与推理/assest/大模型推理框架（三）Text generation inference (TGI)/3.png>)

Tensor parallelism для `Self-Attention` еще проще, поскольку self-attention по своей природе является механизмом multi-head attention: вычисления каждой головы можно распределить на разные GPU.

![alt text](<../../../02-第二章-部署与推理/assest/大模型推理框架（三）Text generation inference (TGI)/4.png>)

На рисунке выше self-attention можно параллельно вычислять на 2 GPU, где каждый GPU вычисляет механизм внимания для одной головы. В принципе, сколько есть голов, столько GPU можно использовать для параллельных вычислений.

2. Continuous batching

В традиционном batch processing весь batch запросов должен полностью завершить обработку, прежде чем результаты будут возвращены вместе. Это означает, что более короткие запросы вынуждены ждать завершения более длинных, что приводит к растрате ресурсов GPU и увеличению latency инференса. Технология Continuous Batching позволяет модели после обработки текущей итерации сразу вернуть результат запроса, если он уже завершен, не дожидаясь завершения всех запросов в batch. Это заметно повышает использование аппаратных ресурсов и сокращает время простоя.

![alt text](<../../../02-第二章-部署与推理/assest/大模型推理框架（三）Text generation inference (TGI)/5.png>)

Кроме того, Continuous Batching помогает решить проблему потерь ресурсов, возникающую из-за разного объема вычислений у разных запросов. За счет планирования на уровне итераций размер batch динамически адаптируется под сложность запросов, что эффективно снижает время ожидания для запросов высокой сложности.

Стоит отметить, что при реализации Continuous Batching необходимо учитывать несколько ключевых вопросов: обработку Early-finished Requests, Late-joining Requests, а также batching запросов разной длины. Две проектные идеи, предложенные OCRA, - Iteration-level Batching и Selective Batching - как раз направлены на решение этих проблем.

3. Flash Attention

Flash Attention - это новый алгоритм механизма внимания, совместно разработанный исследовательскими командами Стэнфордского университета и Университета штата Нью-Йорк в Буффало. Он предназначен для решения проблем высокой временной и memory complexity, с которыми сталкиваются традиционные Transformer-модели при обработке длинных последовательностей. Ключевая идея алгоритма - уменьшить количество операций чтения и записи между GPU high-bandwidth memory (`HBM`) и on-chip SRAM GPU. Благодаря технологиям tiling и recomputation алгоритм значительно снижает частоту обращений к HBM, тем самым повышая скорость выполнения и уменьшая использование памяти.

![alt text](<../../../02-第二章-部署与推理/assest/大模型推理框架（三）Text generation inference (TGI)/6.png>)

Flash Attention за счет IO-aware подхода оптимизирует паттерны доступа к памяти, делает Transformer-модели более эффективными при работе с длинными последовательностями и открывает возможность строить качественные модели с более длинным контекстом.

4. PagedAttention

vLLM представил PagedAttention - алгоритм внимания, вдохновленный классическими идеями виртуальной памяти и paging в операционных системах. В отличие от традиционных алгоритмов внимания, PagedAttention позволяет хранить непрерывные keys и values в несмежных областях памяти. Конкретнее, PagedAttention делит KV cache каждой последовательности на blocks, и каждый block содержит keys и values для фиксированного числа tokens. Во время вычисления внимания kernel PagedAttention эффективно определяет и извлекает эти blocks.

## 2. Возможности TGI

1. **Простой launcher**: TGI предоставляет простой launcher, с помощью которого можно легко обслуживать самые популярные LLMs.

2. **Готовность к production**: TGI интегрирует distributed tracing (с использованием Open Telemetry) и метрики Prometheus, что соответствует требованиям production-сред.

3. **Tensor parallelism**: за счет tensor parallel вычислений на нескольких GPU TGI может заметно ускорять инференс.

4. **Token streaming**: потоковая передача tokens реализована с помощью server-sent events (`SSE`).

5. **Continuous batching**: входящие запросы обрабатываются через continuous batching, что повышает общий throughput.

6. **Оптимизированный код инференса**: для самых популярных архитектур TGI оптимизирует код Transformers с помощью Flash Attention, Paged Attention и других технологий.

7. **Поддержка разных видов quantization**: поддерживаются различные методы quantization, включая bitsandbytes, GPT-Q, EETQ, AWQ, Marlin и fp8.

8. **Безопасная загрузка весов**: для загрузки весов используется Safetensors, что повышает безопасность.

9. **Watermarking**: интегрирована технология watermarking из `"A Watermark for Large Language Models"`.

10. **Гибкое управление генерацией**: поддерживаются logits warper (temperature scaling, top-p, top-k, repetition penalty и др.), stop sequences и вывод log probabilities.

11. **Speculative generation**: реализована оптимизация latency примерно в 2 раза.

12. **Guidance/JSON output**: поддерживается задание формата вывода, что ускоряет инференс и обеспечивает соответствие вывода заданной спецификации.

13. **Custom prompt generation**: вывод модели можно направлять с помощью custom prompt, легко генерируя нужный текст.

14. **Поддержка fine-tuning**: fine-tuned модели можно использовать для выполнения конкретных задач, добиваясь более высокой точности и производительности.

15. **Поддержка hardware**: помимо NVIDIA GPU, TGI поддерживает AMD GPU, Intel GPU, Inferentia, Gaudi, Google TPU и другие аппаратные платформы.

## Введение

В последние годы цены на многие коммерческие большие модели постоянно снижаются, но обращали ли вы внимание на важную деталь: цена output token обычно в несколько раз выше цены input token. В чем причина? В этой статье мы подробно разберем механизм генерации token в больших моделях и покажем, почему стоимость output token выше.

## Механизм вывода большой модели
При поддержке KV cache (кэша ключей-значений) генерация token в большой модели делится на два этапа:

### Предзаполнение (prefill)
На этапе предзаполнения модель параллельно обрабатывает входной prompt (то есть input token) и создает KV cache. Этот шаг включает один полный forward pass (forward) и выводит первый token. Время этого процесса в основном определяется input token, поскольку для инициализации всего процесса генерации нужно выполнить полный расчет.

### Декодирование (decoding)
Этап декодирования — это процесс генерации следующего token по одному. На этом шаге количество output token определяет, сколько раз нужно выполнить forward pass. Хотя каждый forward pass благодаря KV cache выполняется быстрее, модели все равно приходится многократно считать, постепенно генерируя каждый последующий token.

## Факторы влияния
На процесс генерации большой модели влияет множество факторов. Они определяют не только эффективность генерации token, но и стоимость:

### Input Token
1. **Время предзаполнения**: input token определяет время этапа предзаполнения, то есть время генерации первого token; этот процесс вычисляется только один раз.
2. **Нижняя граница потребления видеопамяти**: input token определяет минимальный объем видеопамяти, занимаемый KV cache, потому что они создают основу для последующей генерации token.
3. **Базовая скорость генерации**: input token также задает базовую скорость генерации начиная со второго token и влияет на эффективность дальнейших вычислений.

### Output Token
1. **Длительность декодирования**: output token определяет длительность этапа декодирования. Благодаря KV cache каждое вычисление намного быстрее, чем для первого token, но все равно требуется много forward pass.
2. **Верхняя граница потребления видеопамяти**: output token определяет максимальный объем видеопамяти, занимаемый KV cache, потому что с генерацией каждого нового token объем информации в кэше также растет.

## Какие затраты больше?
Что обходится дороже: один полный forward pass (количество input token) или многократные forward pass с использованием KV cache (количество output token)?

### Бутылочные горлышки вычислений и коммуникации
Хотя KV cache эффективно снижает объем вычислений на каждом шаге, скорость развития пропускной способности коммуникации не успевает за ростом вычислительной мощности, поэтому видеокарты становятся более чувствительными к I/O. Генерация каждого output token все равно требует многократных forward pass, а ограничения скорости I/O видеокарты делают стоимость каждого output token выше. Это также объясняет, почему output token стоит дороже input token.

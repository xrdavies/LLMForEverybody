## 2. Оценка сложности

### 2.1 Уровни сложности

![alt text](<../../../07-第七章-Agent/assest/LLM应用落地指南/3.png>)

- Приложения L1: опираются на собственные возможности LLM, выполняют один вызов LLM и позволяют LLM напрямую сформировать результат;
- Приложения L2: учитывают ограничения собственных возможностей LLM и начинают декомпозировать задачу. Самые ранние RAG-приложения — типичный сценарий этого уровня: они восполняют нехватку знаний LLM через retrieval;
- Приложения L3: вводят такое свойство Agent, как reflection. Reflection — очень важная способность Agent: она позволяет реагировать на информацию из окружения, проверять, достигнута ли цель, и повторно выполнять задачу. Важно: на этом этапе возможности Agent вводятся через код;
- Приложения L4: полноценный Agent. Ему нужно предоставить инструменты, чтобы он мог самостоятельно планировать и с помощью инструментов автономно выполнять любые связанные задачи или отвечать на вопросы.

### 2.2 Оценка сложности: на примере вопросов и ответов по знаниям

Возьмем сценарий вопросов и ответов по знаниям и разберем приложения L1-L4.

**L1: простые вопросы и ответы по знаниям**
- Не предоставляются никакие дополнительные знания, ответы строятся только на собственных возможностях модели;
- Модель могла быть дообучена для вертикальной предметной области.
![alt text](<../../../07-第七章-Agent/assest/LLM应用落地指南/4.png>)

**L2: вопросы и ответы по знаниям на основе RAG**
- Вводится RAG: база знаний восполняет пробелы в знаниях модели и помогает избегать hallucinations.
![alt text](<../../../07-第七章-Agent/assest/LLM应用落地指南/5.png>)

**L3: вопросы и ответы по знаниям с простым Agent**
- Вводится способность к reflection относительно цели. В RAG Agent должен определить, достаточно ли ясно сформулирован вопрос пользователя, иначе нужно уточнение;
- Также нужно определить, нашел ли RAG retrieval знания, которые позволяют ответить на вопрос; если нет, retrieval выполняется снова.
![alt text](<../../../07-第七章-Agent/assest/LLM应用落地指南/6.png>)

**L4: вопросы и ответы по знаниям с полноценным Agent**
- Вводится long-term memory: на основе привычек пользователя можно понять, что он хотел выразить, и проактивно снять неоднозначность в его вопросе;
- Вводится planning: Agent может планировать, в какой из нескольких баз знаний сначала идти за данными, а затем, опираясь на найденные данные, из каких баз знаний получать детальную информацию, повторяя этот цикл итеративно.
![alt text](<../../../07-第七章-Agent/assest/LLM应用落地指南/7.png>)

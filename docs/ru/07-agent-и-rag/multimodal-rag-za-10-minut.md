Умеете делать RAG только по документам? За 10 минут разберемся, как делать мультимодальный RAG

- Подсказка к чтению: для этой статьи нужно базовое понимание технологии RAG. Если вы не знакомы с RAG, можете обратиться к моей предыдущей статье:

Многие RAG-системы строятся на тексте, но в реальных сценариях у нас много изображений и даже видео. Иногда нам также хочется построить мультимодальный RAG для изображений/видео, чтобы можно было задавать вопросы по изображениям/видео.

## 1. Архитектура

Видео можно рассматривать как набор множества кадров-изображений, поэтому начнем с изображений.

![alt text](../../../07-第七章-Agent/assest/10分钟了解如何进行多模态RAG/1.PNG)

Обратите внимание на схему выше: фиолетовым показан процесс построения Vector DB, голубым — процесс inference. Вся мультимодальная архитектура выглядит так:
1. Обработать мультимодальные данные: получить image-text pair.
2. Преобразовать данные image-text pair в embedding и сохранить их в Vector DB.
3. Преобразовать query в embedding и выполнить retrieval в Vector DB.
4. Объединить возвращенный image-text pair с query и получить ответ с помощью модели LVLM.

Можно заметить, что эта архитектура почти полностью совпадает с традиционной архитектурой RAG, просто входы и выходы на каждом шаге отличаются.

![alt text](../../../07-第七章-Agent/assest/10分钟了解如何进行多模态RAG/0.PNG)

## 2. Обработка данных

Нам нужно обработать данные в пары «изображение-текст», чтобы затем удобно передать их в Embedding model.

Для видео возможны 3 ситуации:

- case1: у видео есть субтитры
- case2: у видео нет субтитров
- case3: у видео нет субтитров, и субтитры невозможно сгенерировать, например это немой фильм или видео только с фоновой музыкой

Для изображений возможны две ситуации:

- изображение содержит поясняющий текст
- изображение не содержит поясняющего текста

Методы обработки видео:
1. case1: используем некоторые инструменты, например CV2, нарезаем видео по времени, выбираем подходящие изображения и сопоставляем субтитры с подходящими изображениями.
2. case2: сначала используем инструмент speech-to-text, например whisper, чтобы сгенерировать субтитры, затем применяем схему из case1.
3. case3: генерируем изображения способом из case1, а затем используем Large Vision-Language Model (LVLM), чтобы сгенерировать субтитры.

![alt text](../../../07-第七章-Agent/assest/10分钟了解如何进行多模态RAG/2.png)

Методы обработки изображений:

1. case1: никаких действий не требуется.
2. case2: используем Large Vision-Language Model (LVLM), чтобы сгенерировать описание.

## 3. Embedding

![alt text](../../../07-第七章-Agent/assest/10分钟了解如何进行多模态RAG/3.png)

Мы можем передать описанные выше пары «изображение-текст» в специальную embedding-модель, например BridgeTower. Так можно получить embedding для этого pair, что удобно для последующего retrieval. Подробнее о модели BridgeTower см. в этой статье: [bridge tower](https://arxiv.org/abs/2206.08657)

![alt text](../../../07-第七章-Agent/assest/10分钟了解如何进行多模态RAG/4.png)

## 4. Retrieval, Prompt process & Response

После retrieval по пользовательскому query мы получим image-text pair. Затем обрабатываем query, text и image, подаем их в упомянутую выше LVLM и получаем response.

![alt text](../../../07-第七章-Agent/assest/10分钟了解如何进行多模态RAG/5.png)

LLaVA — часто используемая LVLM, см. https://arxiv.org/pdf/2304.08485.

Можно видеть, что мультимодальный RAG пытается приблизиться к традиционному RAG; просто embedding и LM для генерации ответа используются мультимодальные.

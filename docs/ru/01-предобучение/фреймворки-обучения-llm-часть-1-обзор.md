# Фреймворки обучения LLM, часть 1: обзор

## 1. Введение

Хотя PyTorch - очень популярный фреймворк deep learning, у него есть множество преимуществ: dynamic computation graph, простота использования, гибкость и сильная поддержка сообщества. Но для обучения LLM нам нужны более специализированные фреймворки, чтобы закрывать особые требования. Главная причина - `поддержка распределенного обучения`.

`Поддержка распределенного обучения`: крупные модели обычно требуют распределенного обучения на нескольких GPU или TPU. Хотя PyTorch поддерживает распределенное обучение, фреймворки обучения LLM могут предоставлять более оптимизированные стратегии распределенного обучения, например model parallelism и data parallelism.

## 2. Краткое введение в технологии параллелизации

В обучении LLM мы обычно используем несколько следующих технологий параллелизации:

1. DataParallel (DP) - одна и та же конфигурация копируется несколько раз, и в каждую копию подается часть данных. Обработка идет параллельно, а все копии синхронизируются в конце каждого training step.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（一）：综述/0.png)

2. TensorParallel (TP) - каждый tensor делится на несколько блоков: вместо того чтобы хранить весь tensor на одном GPU, каждый shard tensor размещается на своем GPU. Во время обработки каждый shard отдельно и параллельно обрабатывается на разных GPU, а результаты синхронизируются в конце шага. Это называют горизонтальным параллелизмом, потому что разбиение происходит на горизонтальном уровне.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（一）：综述/2.png)

3. PipelineParallel (PP) - модель вертикально, по слоям, разделяется между несколькими GPU, поэтому на одном GPU размещается только один слой или несколько слоев модели. Каждый GPU параллельно обрабатывает разные стадии pipeline и работает с небольшой частью batch.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（一）：综述/4.png)

4. MixedParallel (MP) - подход, который объединяет data parallelism, model parallelism и pipeline parallelism. Он позволяет одновременно выполнять data parallelism и model parallelism на нескольких GPU, а также использовать pipeline parallelism внутри каждого GPU.

## 3. Фреймворки обучения

Сейчас существует много фреймворков обучения больших моделей, которые помогают ускорить обучение крупных моделей. Обычно такие фреймворки предоставляют более оптимизированные стратегии распределенного обучения, например model parallelism и data parallelism. Я выбрал четыре наиболее влиятельных mainstream-фреймворка:

1. FSDP

Fully Sharded Data Parallel (FSDP)[1](#refer-anchor-1) - это метод data parallelism, впервые предложенный FairScale-FSDP в 2021 году и позднее интегрированный в PyTorch 1.11.

FSDP можно рассматривать как реализацию `ZERO-3` из трех уровней алгоритма ZERO, предложенных во фреймворке Microsoft DeepSpeed. За счет sharding градиентов модели, optimizer states и параметров каждый GPU хранит только часть информации о параметрах, что оптимизирует использование ресурсов и повышает эффективность обучения. Кроме того, FSDP тесно совместно спроектирован с несколькими ключевыми core-компонентами PyTorch, включая Tensor implementation, scheduler system и CUDA memory caching allocator, чтобы обеспечить неинвазивный пользовательский опыт и высокую эффективность обучения.

2. DeepSpeed

DeepSpeed[2](#refer-anchor-2) - библиотека оптимизации deep learning, разработанная исследовательской командой Microsoft. Она предназначена для эффективного и масштабируемого обучения крупных моделей. DeepSpeed значительно повышает эффективность обучения и снижает требования к ресурсам за счет продвинутых стратегий параллелизации, технологий оптимизации памяти, таких как ZeRO memory optimizer, и mixed precision training.

3. Megatron-LM

Megatron-LM[3](#refer-anchor-3) - распределенный фреймворк обучения больших языковых моделей от NVIDIA. Он поддерживает обучение моделей в multi-node, multi-GPU окружении. Megatron-LM через model parallelism позволяет обучать модели с сотнями миллиардов параметров. Этот фреймворк комплексно использует Data Parallelism, Tensor Parallelism и Pipeline Parallelism для обучения больших моделей вроде GPT-3.

`Небольшой tips`: **transformer** также может означать **трансформер**, а **megatron** - это **Мегатрон**.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（一）：综述/5.png)

4. Accelerate

Accelerate[4](#refer-anchor-4) от Hugging Face - библиотека для упрощения и ускорения обучения deep learning моделей. Она поддерживает распределенное обучение на разных аппаратных конфигурациях, включая CPU, GPU, TPU и т. д. Accelerate позволяет пользователям легко переключаться между разными стратегиями параллелизации, а также поддерживает mixed precision training, что может дополнительно повысить эффективность обучения.

Ниже показана поддержка стратегий параллелизации разными фреймворками по состоянию на 2024/10/12:

| Фреймворк | DP | PP | TP | 3D parallelism |
| :--- |:----:| :----: |:---: |:---: |
| PyTorch(FSDP) | да | нет | нет | нет |
| DeepSpeed | да | да | да | да |
| Megatron-LM | да | да | да | да |
| Accelerate | да | нет | нет | нет |

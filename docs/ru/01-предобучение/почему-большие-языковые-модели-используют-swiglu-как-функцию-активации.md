Почему большие языковые модели используют SwiGLU как функцию активации?

Функции активации — ключевая часть проектирования нейронных сетей. Они вводят нелинейность и управляют потоком градиентов, благодаря чему сеть может обучаться и выполнять разные сложные задачи. В больших языковых моделях использование SwiGLU как функции активации уже стало заметным трендом. Так почему большие языковые модели используют SwiGLU как функцию активации?

**Совет по чтению**: можно смотреть только формулы и код, чтобы понять математическое выражение и способ реализации функции активации.


## 1. Самая ранняя функция активации — Sigmoid

В биологии активация нейронов обычно обладает нелинейностью и пороговым характером: нейрон активируется и выдает выходной сигнал только после получения достаточно сильного входного сигнала. В искусственных нейронных сетях, хотя использование функции sigmoid в глубоком обучении уже сократилось, когда-то она была одной из самых популярных функций активации. S-образная кривая функции sigmoid в некоторой степени похожа на способ активации биологического нейрона: у нее есть явный порог, и когда вход превышает этот порог, выход резко возрастает.

![alt text](<../../../01-第一章-预训练/assest/为什么大型语言模型都在使用 SwiGLU 作为激活函数？/0.png>)

Функция Sigmoid — широко применяемая в статистике и машинном обучении функция активации, особенно в ранних моделях нейронных сетей. Она имеет S-образную форму и сжимает входное значение в диапазон от 0 до 1. Поэтому в некоторых случаях она особенно полезна, например когда выход нужно интерпретировать как вероятность или использовать в качестве функции активации выходного слоя в задаче бинарной классификации.

Математическое выражение функции Sigmoid:

$$ \sigma(x) = \frac{1}{1 + e^{-x}} $$

где $e$ — основание натурального логарифма (примерно 2.71828), а $x$ — входное значение.

![alt text](<../../../01-第一章-预训练/assest/为什么大型语言模型都在使用 SwiGLU 作为激活函数？/1.png>)

### Свойства функции Sigmoid включают:

1. **Диапазон выхода**: выходное значение всегда находится в интервале (0, 1), что делает функцию sigmoid очень полезной при выводе вероятностей или в задачах бинарной классификации.

2. **Гладкость и непрерывность**: функция sigmoid гладкая и непрерывная на всей области определения, что помогает вычислять градиенты в процессе оптимизации.

3. **Центральная симметрия**: функция sigmoid симметрична относительно прямой $y = x$, то есть она может обрабатывать положительные и отрицательные входные значения.

4. **Проблема градиента**: когда входное значение $x$ очень большое или очень маленькое, градиент функции sigmoid близок к 0. Это может привести к проблеме исчезающего градиента, из-за чего в глубоких слоях нейронной сети градиенту трудно распространяться.

5. **Не нуль-центрированность**: выход функции sigmoid не центрирован относительно 0, из-за чего сети может потребоваться больше времени и ресурсов на корректировку весов в процессе оптимизации.

### Применения функции Sigmoid:

- **Бинарная классификация**: в выходном слое задачи бинарной классификации функция sigmoid может преобразовывать выход в вероятность принадлежности к определенному классу.
- **Логистическая регрессия**: в модели логистической регрессии функция sigmoid используется для преобразования выхода линейной регрессии в вероятностный прогноз.
- **Скрытые слои**: хотя в современных моделях глубокого обучения ее реже используют в скрытых слоях, в некоторых случаях sigmoid все еще может выступать функцией активации скрытого слоя.

### Ограничения:

- **Исчезающий градиент**: в глубоких сетях, поскольку при больших по модулю входных значениях градиент функции sigmoid близок к 0, легко возникает проблема исчезающего градиента, что мешает обучению глубоких сетей.
- **Вычислительные затраты**: по сравнению с некоторыми функциями активации, например ReLU, вычислительная сложность sigmoid выше, поскольку она включает экспоненту.

Из-за этих ограничений в современных моделях глубокого обучения функция sigmoid постепенно была заменена другими функциями активации, такими как ReLU и ее варианты, особенно в скрытых слоях. Однако в отдельных сценариях, где нужен вероятностный прогноз на выходе, функция sigmoid остается важной функцией активации.

```python
import numpy as np
def sigmoid(x):
    """Sigmoid 激活函数

    参数:
    x -- 输入值

    返回:
    Sigmoid 激活后的值
    """
    return 1 / (1 + np.exp(-x))

```


## 2. Герой глубокого обучения — ReLU

ReLU (Rectified Linear Unit) — простая и эффективная функция активации, широко применяемая в глубоком обучении. Функция ReLU определяется так:

$$ f(x) = max(0, x) $$

Это означает, что когда вход $x$ больше 0, выход равен входу; когда вход $x$ меньше или равен 0, выход равен 0. График функции ReLU — ломаная линия: при положительном $x$ наклон функции равен 1, а при отрицательном $x$ наклон равен 0.

![alt text](<../../../01-第一章-预训练/assest/为什么大型语言模型都在使用 SwiGLU 作为激活函数？/2.png>)

Свойства ReLU включают:
- Разреженная активация: поскольку на отрицательной полуоси выход ReLU равен 0, это означает, что в любой момент примерно половина нейронов (если предположить, что вход имеет нулевое среднее) не будет активирована. Такая разреженность помогает уменьшить объем вычислений и предотвратить переобучение.

- Нелинейность: ReLU вводит нелинейность, благодаря чему нейронная сеть может изучать и моделировать сложные функции.

- Высокая вычислительная эффективность: благодаря простой математической форме ReLU вычисляется очень быстро, что помогает ускорять обучение и inference нейронных сетей.

- Смягчение проблемы исчезающего градиента: по сравнению с функциями sigmoid или tanh, градиент ReLU на положительном участке постоянно равен 1. Это помогает смягчить проблему исчезающего градиента и делает обучение глубоких сетей более осуществимым.

```python
import numpy as np
def relu(x):
    """ReLU 激活函数

    参数:
    x -- 输入值

    返回:
    ReLU 激活后的值
    """
    return np.maximum(0, x)
```


## 3. Расцвет вариантов — ReLU-подобные функции активации

Помимо ReLU существует множество похожих функций активации. Они улучшают ReLU, чтобы решить часть присущих ей проблем. К таким ReLU-подобным функциям активации относятся:

- **Leaky ReLU**: Leaky ReLU — улучшение ReLU. Когда вход отрицательный, Leaky ReLU вводит небольшой наклон, а не сразу выдает 0. Это помогает решить проблему нулевого выхода ReLU на отрицательном участке.

- **Parametric ReLU (PReLU)**: PReLU — расширение Leaky ReLU, которое вводит обучаемый параметр для управления наклоном на отрицательном участке. Благодаря этому нейронная сеть может адаптивно обучать функцию активации на отрицательном участке.

- **Exponential Linear Unit (ELU)**: ELU — ReLU-подобная функция активации, которая вводит экспоненциальный член на отрицательном участке, делая выход на этом участке более гладким. В некоторых случаях ELU может давать лучшее качество.

- **Scaled Exponential Linear Unit (SELU)**: SELU — вариант ELU, который вводит масштабирующие параметры. При определенных условиях SELU может сохранять среднее и дисперсию входа неизменными, что помогает стабилизировать обучение.

- **GELU**: GELU — функция активации на основе Gaussian Error Linear Unit. В некоторых случаях она может давать лучшее качество. Математическая форма GELU — гладкая функция с функцией ошибок.

На практике эти ReLU-подобные функции активации получили широкое применение. В определенной степени они улучшают качество ReLU, а в разных сценариях могут проявлять себя по-разному. Выбор подходящей функции активации зависит от конкретной задачи и архитектуры модели.


## 4. Лучшая функция активации по экспериментам — Swish

Swish — функция активации, предложенная Google Brain. Ее математическое выражение:
$$ f(x) = x \cdot \sigma(x) $$
где $\sigma(x)$ — функция sigmoid.

![alt text](<../../../01-第一章-预训练/assest/为什么大型语言模型都在使用 SwiGLU 作为激活函数？/3.png>)

Название Swish, возможно, связано с тем, что ее форма похожа на рыбий хвост и вызывает ассоциацию с гладким, плавным движением, что соответствует смыслу слова "swish".

Свойства функции Swish включают:
- **Нелинейность**: Swish вводит нелинейность, благодаря чему нейронная сеть может изучать и моделировать сложные функции.
- **Гладкость**: функция Swish гладкая и непрерывная на всей области определения, что помогает вычислять градиенты в процессе оптимизации.
- **Адаптивность**: выход функции Swish зависит от входного значения, поэтому она может адаптивно менять форму функции активации.

В некоторых экспериментах функция Swish показала более высокое качество, чем ReLU, особенно в отдельных глубоких нейронных сетях. Однако вычислительная сложность Swish выше, поскольку она включает вычисление функции sigmoid. Поэтому на практике подходящую функцию активации нужно выбирать с учетом конкретной задачи и архитектуры модели.

```python
import numpy as np
def swish(x,beta=1.0):
    """Swish 激活函数

    参数:
    x -- 输入值

    返回:
    Swish 激活后的值
    """
    return x * sigmoid(beta*x)

def sigmoid(x):
    """Sigmoid 函数

    参数:
    x -- 输入值

    返回:
    Sigmoid 函数的输出值
    """
    return 1 / (1 + np.exp(-x))
```


## 5. GLU

GLU (Gated Linear Unit) на самом деле не совсем функция активации, а слой нейронной сети. Это структура, где за линейным преобразованием следует gating-механизм. В нем функция sigmoid управляет тем, сколько информации может пройти. GLU объединяет Linear Unit и gating-механизм, позволяя эффективно изучать разные признаки входных данных. Математическое выражение GLU:

$$f(X) = (X ∗ W + b) ⊗ σ(X ∗ V + c)$$
где ⊗ обозначает поэлементное умножение, $X$ — вход, $W$ и $V$ — матрицы весов, $b$ и $c$ — смещения.

Свойства GLU включают:
- **Gating-механизм**: GLU вводит gating-механизм и с помощью функции sigmoid управляет линейным преобразованием входа, благодаря чему нейронная сеть может изучать разные признаки входных данных.
- **Нелинейность**: GLU вводит нелинейность, благодаря чему нейронная сеть может изучать и моделировать сложные функции.
- **Адаптивность**: выход функции GLU зависит от входного значения, поэтому она может адаптивно менять форму функции активации.

Реализация GLU в pytorch:

![alt text](<../../../01-第一章-预训练/assest/为什么大型语言模型都在使用 SwiGLU 作为激活函数？/4.png>)

```python
import numpy as np
def glu(x):
    """GLU 激活函数

    参数:
    x -- 输入数组，维度必须是偶数

    返回:
    GLU 激活后的数组
    """
    assert x.shape[-1] % 2 == 0, "输入数组的最后一个维度必须是偶数"
    half_dim = x.shape[-1] // 2
    return x[..., :half_dim] * sigmoid(x[..., half_dim:])

def sigmoid(x):
    """Sigmoid 函数

    参数:
    x -- 输入值

    返回:
    Sigmoid 函数的输出值
    """
    return 1 / (1 + np.exp(-x))
```

## 6. Практика рождает истину — SwiGLU

Наконец мы добрались до главного героя сегодняшней статьи — SwiGLU. SwiGLU — функция активации, объединяющая Swish и GLU. Она сочетает гладкость Swish и gating-механизм GLU, позволяя эффективно изучать разные признаки входных данных. Математическое выражение SwiGLU:

$$ f(X) = (X ∗ W + b) ⊗ Swish(X ∗ V + c) $$

```python
import numpy as np
def SwiGLU(x):
    """SwiGLU 激活函数

    参数:
    x -- 输入数组，维度必须是偶数

    返回:
    SwiGLU 激活后的数组
    """
    assert x.shape[-1] % 2 == 0, "输入数组的最后一个维度必须是偶数"
    half_dim = x.shape[-1] // 2
    return x[..., :half_dim] * swish(x[..., half_dim:])

def swish(x,beta=1.0):
    """Swish 激活函数

    参数:
    x -- 输入值

    返回:
    Swish 激活后的值
    """
    return x * sigmoid(beta*x)

def sigmoid(x):
    """Sigmoid 函数

    参数:
    x -- 输入值

    返回:
    Sigmoid 函数的输出值
    """
    return 1 / (1 + np.exp(-x))

```

## Почему ее используют?

Функция активации SwiGLU широко применяется в больших языковых моделях благодаря своим преимуществам в нескольких аспектах. Она объединяет свойства Swish и GLU и дает эффективный механизм активации. В частности:

1. **Нелинейная способность**: SwiGLU вводит нелинейность через функцию активации Swish, что позволяет модели изучать и представлять более сложные паттерны данных.

2. **Gating-свойства**: gating-механизм GLU позволяет модели динамически настраивать информационный поток, благодаря чему при обработке длинных последовательностей модель лучше улавливает дальние зависимости.

3. **Стабильность градиентов**: SwiGLU дает ненулевой градиент в области отрицательных входов, помогая смягчить проблему исчезающего градиента и тем самым повысить стабильность обучения модели.

4. **Обучаемые параметры**: параметры SwiGLU могут обучаться в процессе training, поэтому модель может динамически адаптироваться под разные задачи и наборы данных, повышая гибкость и приспособляемость.

5. **Вычислительная эффективность**: по сравнению с некоторыми сложными функциями активации SwiGLU сохраняет качество и при этом обладает высокой вычислительной эффективностью, что особенно важно для обучения и inference крупномасштабных языковых моделей.

Благодаря этим преимуществам SwiGLU применяется в больших языковых моделях, таких как LLAMA, OLMO и PALM. Сочетая гладкость Swish и gating-механизм GLU, она дает эффективную функцию активации, поддерживающую обучение сложных и эффективных моделей глубокого обучения.

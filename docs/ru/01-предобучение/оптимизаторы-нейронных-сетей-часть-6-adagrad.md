## 1. Появление алгоритма AdaGrad
AdaGrad (Adaptive Gradient Algorithm) был предложен John Duchi, Elad Hazan и Yoram Singer. Алгоритм подробно описан в статье 2011 года «Adaptive Subgradient Methods for Online Learning and Stochastic Optimization» [1](#refer-anchor-6), опубликованной в Journal of Machine Learning Research. Главная особенность AdaGrad состоит в том, что он независимо настраивает learning rate для каждого параметра: редко обновляемые параметры получают больший learning rate, а часто обновляемые - меньший. Такой способ адаптивной настройки learning rate особенно хорошо подходит для обработки разреженных данных, поскольку позволяет уделять больше внимания разреженным признакам. Однако у AdaGrad есть и недостаток: накопленная в процессе обучения сумма квадратов градиентов может привести к тому, что learning rate станет слишком маленьким, и на поздних этапах обучение почти остановится. Чтобы решить эту проблему, последующие исследователи предложили варианты AdaGrad, такие как AdaDelta и Adam.

## 2. Принцип работы алгоритма AdaGrad

1. **Инициализация**: для каждого параметра $\theta_i$ инициализировать сумму квадратов градиентов $\sum g_i^2 = 0$.

2. **Вычисление градиента**: на каждой итерации вычислить градиент $g_i$ параметра $\theta_i$.

3. **Обновление суммы квадратов градиентов**:
   $\sum g_i^2 = \sum g_i^2 + g_i^2$

4. **Вычисление адаптивного learning rate**:
   $\eta_i = \frac{\eta}{\sqrt{\sum g_i^2} + \epsilon}$
   где $\eta$ - глобальный learning rate, а $ \epsilon $ - очень малое число (например, $1e-8$), используемое для предотвращения деления на ноль.

5. **Обновление параметра**:
   $\theta_i = \theta_i - \eta_i \cdot g_i$

### Параметры:

- **$\eta$**: глобальный learning rate, управляющий начальной скоростью обучения.
- **$\epsilon$**: малая константа для численной стабильности, предотвращающая деление на ноль.


AdaGrad (Adaptive Gradient Algorithm) - это алгоритм градиентного спуска для оптимизации крупномасштабных алгоритмов машинного обучения. Он устраняет некоторые ограничения стандартного градиентного спуска за счет адаптивной настройки learning rate для каждого параметра, особенно при работе с разреженными данными.

## 3. Основные особенности AdaGrad:

1. **Адаптивный learning rate**: AdaGrad задает learning rate отдельно для каждого параметра. Это значит, что learning rate каждого параметра может корректироваться на основе истории его градиентов.

2. **Обработка разреженных данных**: AdaGrad особенно хорошо подходит для разреженных данных, поскольку уменьшает learning rate для часто обновляемых параметров и увеличивает его для редко обновляемых.

3. **Не требует ручной настройки learning rate**: AdaGrad не требует вручную задавать learning rate, он автоматически настраивает его на основе истории обновлений параметров.

## 4. Преимущества и ограничения:

**Преимущества**:
- Адаптивный learning rate, хорошо подходит для обработки разреженных данных.
- Не требует ручной настройки learning rate.

**Ограничения**:
- Learning rate уменьшается, что может привести к ранней остановке обучения, особенно при работе с невыпуклыми задачами.
- Для очень больших наборов данных накопленная сумма квадратов градиентов может стать очень большой, из-за чего learning rate окажется слишком маленьким.

AdaGrad - эффективный алгоритм оптимизации, особенно для крупномасштабных и разреженных наборов данных. Однако из-за свойства убывающего learning rate его может потребоваться сочетать с другими алгоритмами оптимизации, такими как RMSProp или Adam, чтобы преодолеть ограничения.

Фреймворки обучения LLM (часть 3): DeepSpeed

DeepSpeed[1](#refer-anchor-1) — это библиотека оптимизации глубокого обучения, разработанная исследовательской командой Microsoft. Она предназначена для эффективного и масштабируемого обучения больших моделей. DeepSpeed значительно повышает эффективность обучения и снижает требования к ресурсам за счет продвинутых стратегий параллелизма, методов оптимизации памяти (например, оптимизатора памяти ZeRO) и обучения со смешанной точностью.

## 1. ZeRO

ZeRO (Zero Redundancy Optimizer) — одна из ключевых технологий DeepSpeed. Это оптимизатор, спроектированный для решения проблемы узкого места по памяти при крупномасштабном распределенном обучении. ZeRO существенно снижает требуемый объем памяти, оптимизируя хранение и коммуникацию состояний модели, что позволяет обучать более крупные модели при ограниченных ресурсах. DeepSpeed — это open-source библиотека оптимизации глубокого обучения от Microsoft, цель которой повысить эффективность и масштабируемость обучения больших моделей, а ZeRO является одним из ее основных компонентов: он оптимизирует использование памяти и позволяет обучать модели большего размера.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（三）DeepSpeed/0.png)

ZeRO делится на три уровня оптимизации: ZeRO-1, ZeRO-2 и ZeRO-3. Каждый следующий уровень дополнительно снижает потребление памяти относительно предыдущего.

1. **ZeRO-1**: на этом этапе состояния оптимизатора (например, веса и градиенты оптимизатора Adam) распределяются между несколькими GPU, а не хранятся полностью на каждом GPU. Это экономит часть видеопамяти, но параметры модели и активации по-прежнему должны полностью храниться на каждом GPU.

2. **ZeRO-2**: поверх ZeRO-1 дополнительно выполняется шардирование градиентов. Помимо состояний оптимизатора, градиенты также распределяются между несколькими GPU. Это еще больше уменьшает использование памяти на каждом GPU и повышает вычислительную эффективность.

3. **ZeRO-3**: на этом этапе реализовано полное шардирование всех состояний модели, включая параметры модели. Это означает, что параметры модели, состояния оптимизатора и градиенты распределяются между несколькими GPU. Такой подход позволяет при том же объеме видеопамяти обучать более крупные модели, но может увеличить коммуникационные накладные расходы.

Кроме того, существует ZeRO-Infinity — расширение ZeRO-3, которое может использовать память CPU и NVMe для дальнейшего расширения памяти GPU и поддерживает обучение еще более крупных моделей.

FSDP можно понимать как реализацию ZeRO-3: он шардирует градиенты модели, состояния оптимизатора и параметры, так что каждый GPU хранит только часть информации о параметрах. Это оптимизирует использование ресурсов и повышает эффективность обучения.

## 2. DeepSpeed: стратегии параллелизма

DeepSpeed поддерживает несколько стратегий параллелизма, включая параллелизм данных и параллелизм модели (в том числе конвейерный и тензорный параллелизм). Эти методы можно гибко комбинировать, адаптируя их под модели глубокого обучения разного масштаба и сложности.

**Параллелизм данных** (Data Parallelism) распределяет копии модели по нескольким GPU: каждый GPU обрабатывает свой поднабор данных, а затем в конце каждого шага обучения параметры модели синхронизируются. Этот метод подходит для случаев, когда модель достаточно крупная и не может полностью поместиться в память одного GPU. Параллелизм данных в основном использует описанную выше стратегию ZeRO.

**Конвейерный параллелизм** (Pipeline Parallelism) делит слои модели на несколько стадий, которые могут параллельно обрабатываться на разных процессорах. Этот метод повышает эффективность использования памяти и вычислений и особенно хорошо подходит для обучения глубоких нейронных сетей.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（三）DeepSpeed/2.png)

Обучающие данные каждого батча делятся на меньшие микробатчи (micro-batches), которые могут параллельно обрабатываться на разных стадиях конвейера. Как только одна стадия завершает прямой проход для микробатча, память активаций передается следующей стадии конвейера. Аналогично, когда следующая стадия завершает обратный проход для микробатча, градиенты относительно активаций передаются назад по конвейеру. Каждый обратный проход локально накапливает градиенты, после чего все группы параллелизма данных параллельно выполняют редукцию градиентов. В конце оптимизатор обновляет веса модели.

**Тензорный параллелизм (Tensor Parallelism)** разделяет тензоры параметров модели между несколькими GPU. Это позволяет сохранить общую структуру модели и ускорить обучение с помощью распределенных вычислений. Тензорный параллелизм обычно используют, когда число параметров модели настолько велико, что один GPU не может вместить всю модель.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（三）DeepSpeed/4.png)

Главное различие между конвейерным и тензорным параллелизмом в DeepSpeed заключается в том, как они делят модель. Конвейерный параллелизм делит ее по слоям, а тензорный параллелизм — по тензорам параметров. Эти два вида параллелизма можно использовать совместно, формируя стратегию гибридного параллелизма для дальнейшего повышения эффективности обучения и масштабируемости. Например, внутри каждой стадии конвейера можно использовать тензорный параллелизм, чтобы разделить параметры внутри слоя и добиться более мелкозернистого параллелизма.

## 3. Реализация DeepSpeed в PyTorch:

Использование DeepSpeed для обучения моделей глубокого обучения в PyTorch в основном включает следующие шаги:

1. **Установить DeepSpeed**:
   - Установить DeepSpeed через `pip`: `pip install deepspeed`.

2. **Подготовить конфигурационный файл**:
   - Создать конфигурационный файл `deepspeed_config.json`, определяющий параметры обучения и настройки модели. Например:
     ```json
     {
       "train_batch_size": 4,
       "optimizer": {
         "type": "SGD",
         "params": {
           "lr": 0.001,
           "momentum": 0.9
         }
       },
       "fp16": {
         "enabled": true
       },
       "zero_optimization": {
         "stage": 2
       }
     }
     ```

3. **Написать обучающий скрипт**:
   - Импортировать библиотеку DeepSpeed: `import deepspeed`.
   - Определить модель, загрузчик данных и оптимизатор.
   - Использовать функцию `deepspeed.initialize()` для инициализации движка DeepSpeed, обернув модель и оптимизатор:
     ```python
     model_engine, optimizer, _, _ = deepspeed.initialize(args=cmd_args,
                                                           model=model,
                                                           model_parameters=params)
     ```

4. **Обучать модель**:
   - Заменить исходный цикл обучения вызовами `model_engine.backward(loss)` и `model_engine.step()` для выполнения обратного распространения и обновления параметров.
   - DeepSpeed автоматически обрабатывает накопление градиентов, сжатие градиентов и другие техники для повышения эффективности обучения.

5. **Сохранять и загружать checkpoints**:
   - Использовать `model_engine.save_checkpoint()` и `model_engine.load_checkpoint()` для сохранения и загрузки checkpoints модели.

6. **Запустить обучение**:
   - Использовать CLI-инструмент DeepSpeed для запуска распределенного обучения. Например:
     ```
     deepspeed --hostfile=myhostfile --no_ssh --node_rank=<n> \
     --master_addr=<addr> --master_port=<port> \
     <client_entry.py> <client args> \
     --deepspeed --deepspeed_config ds_config.json
     ```
   - В одноузловой многопроцессорной среде с несколькими GPU можно использовать параметры `--include` и `--exclude`, чтобы выбрать используемые GPU.

7. **Мониторинг и настройка**:
   - Во время обучения использовать инструменты DeepSpeed для мониторинга производительности и тюнинга.

8. **Обучение со смешанной точностью**:
   - Включить обучение со смешанной точностью в конфигурационном файле, например установив `"fp16": {"enabled": true}`.

9. **Технология оптимизации ZeRO**:
   - Задать стратегию оптимизации ZeRO в конфигурационном файле, например `"zero_optimization": {"stage": 2}`.

10. **Offload-оптимизация**:
    - При необходимости включить ZeRO-Offload в конфигурационном файле, чтобы выгрузить часть вычислений и памяти на CPU, например `"zero_optimization": {"offload_optimizer": {"device": "cpu", "pin_memory": true}}`.

На момент завершения этой статьи (2024/10/14) поддержка DeepSpeed в PyTorch в основном сосредоточена на ZeRO; поддержка PP и TP ограничена.

## 4. Реализация DeepSpeed в Accelerate:

Библиотека Accelerate предоставляет простой интерфейс для интеграции DeepSpeed, упрощая распределенное обучение в PyTorch. Ниже приведены базовые шаги для распределенного обучения с DeepSpeed и Accelerate:

1. **Установить DeepSpeed и Accelerate**:
   ```bash
   pip install deepspeed accelerate
   ```

2. **Создать конфигурационный файл DeepSpeed**:
   Создать конфигурационный файл `deepspeed_config.json`, определяющий параметры обучения и настройки модели. Например:
   ```json
   {
     "train_batch_size": 4,
     "optimizer": {
       "type": "SGD",
       "params": {
         "lr": 0.001,
         "momentum": 0.9
       }
     },
     "fp16": {
       "enabled": true
     },
     "zero_optimization": {
       "stage": 2
     }
   }
   ```

3. **Написать обучающий скрипт**:
   Импортировать необходимые библиотеки и определить модель, загрузчик данных и оптимизатор. Использовать `Accelerator` и `DeepSpeedPlugin` из Accelerate для подготовки модели, оптимизатора и загрузчика данных. Например:
   ```python
   import torch
   import torch.nn as nn
   from torch.utils.data import TensorDataset, DataLoader
   from accelerate import Accelerator, DeepSpeedPlugin

   class TestNet(nn.Module):
       def __init__(self, input_dim: int, output_dim: int):
           super(TestNet, self).__init__()
           self.fc1 = nn.Linear(in_features=input_dim, out_features=output_dim)
           self.fc2 = nn.Linear(in_features=output_dim, out_features=output_dim)

       def forward(self, x: torch.Tensor):
           x = torch.relu(self.fc1(x))
           x = self.fc2(x)
           return x

   if __name__ == "__main__":
       input_dim = 8
       output_dim = 64
       batch_size = 8
       dataset_size = 1000
       input_data = torch.randn(dataset_size, input_dim)
       labels = torch.randn(dataset_size, output_dim)
       dataset = TensorDataset(input_data, labels)
       dataloader = DataLoader(dataset=dataset, batch_size=batch_size)

       model = TestNet(input_dim=input_dim, output_dim=output_dim)
       accelerator = Accelerator()
       optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
       loss_func = nn.MSELoss()

       model, optimizer, dataloader = accelerator.prepare(model, optimizer, dataloader)

       for epoch in range(10):
           model.train()
           for batch in dataloader:
               inputs, labels = batch
               optimizer.zero_grad()
               outputs = model(inputs)
               loss = loss_func(outputs, labels)
               accelerator.backward(loss)
               optimizer.step()
           print(f"Epoch {epoch}, Loss: {loss.item()}")
   ```

4. **Запустить обучение**:
   Использовать команду `launch` из Accelerate для запуска распределенного обучения. Например:
   ```bash
   accelerate launch --config_file default_config.yaml my_training_script.py
   ```
   Здесь `default_config.yaml` — конфигурационный файл Accelerate, который можно сгенерировать командой `accelerate config`.

5. **Мониторинг и настройка**:
   Во время обучения использовать инструменты DeepSpeed для мониторинга производительности и тюнинга.

6. **Сохранять и загружать checkpoints**:
   Использовать методы `save` и `load` из Accelerate для сохранения и загрузки checkpoints модели.

На момент завершения этой статьи (2024/10/14) поддержка DeepSpeed в Accelerate в основном сосредоточена на ZeRO; в Accelerate пока нет PP и TP.

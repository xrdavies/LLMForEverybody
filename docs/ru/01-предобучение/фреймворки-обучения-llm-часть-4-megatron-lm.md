Фреймворки обучения LLM (часть 4): Megatron-LM

Megatron-LM[1](#refer-anchor-1) — это распределенный фреймворк обучения больших языковых моделей от NVIDIA. Он поддерживает обучение моделей в многоузловых средах с несколькими GPU. Megatron-LM использует параллелизм модели (Model Parallelism), позволяя обучать модели с сотнями миллиардов параметров. Фреймворк комплексно применяет параллелизм данных (Data Parallelism), тензорный параллелизм (Tensor Parallelism) и конвейерный параллелизм (Pipeline Parallelism) для обучения крупных моделей вроде GPT-3.

`Небольшой tips`: **transformer** также может означать **трансформер**, а **megatron** — это **Мегатрон**.

## 1. Особенности Megatron-LM

Megatron-LM воспроизводит крупные модели, такие как GPT-3, с помощью параллелизма данных, тензорного параллелизма и конвейерного параллелизма. Он комплексно применяет несколько стратегий параллелизма для обучения больших языковых моделей, включая тензорный параллелизм (Tensor Parallelism), конвейерный параллелизм (Pipeline Parallelism) и параллелизм данных (Data Parallelism). Эти технологии помогают решать проблемы ограничений видеопамяти, вычислительной сложности и выбора стратегий параллелизма, делая возможным обучение более крупных моделей при ограниченных аппаратных ресурсах.

![alt text](../../../01-第一章-预训练/assest/大模型训练框架（四）Megatron-LM/1.png)

Особенности Megatron-LM включают:

1. **Загрузка данных**: в нем есть эффективный DataLoader, где данные перед обучением tokenize и shuffle. Он также разбивает данные на пронумерованные последовательности с индексами и сохраняет индексы, поэтому tokenize нужно вычислять только один раз;

2. **Слитые CUDA-ядра**: Megatron-LM использует fused-реализацию AdamW из Apex, которая быстрее реализации PyTorch. Идея fused-ядер состоит в том, чтобы объединять похожие операции, которые PyTorch обычно выполняет отдельно, в одну аппаратную операцию, уменьшая число перемещений памяти между множеством отдельных вычислений;

3. **Технологии параллелизма модели**: поддерживает разные стратегии параллелизма, включая тензорный параллелизм, последовательностный параллелизм, конвейерный параллелизм, контекстный параллелизм и MoE expert parallelism;

4. **Оптимизатор и activation checkpointing**: встроенный распределенный оптимизатор и функция повторного вычисления активаций обеспечивают эффективность и стабильность процесса обучения;

5. **FlashAttention**: эффективная реализация механизма внимания, которая заметно повышает скорость обучения;

6. **Поддержка мультимодальности**: в последних версиях добавлена поддержка мультимодального обучения, расширяющая сценарии применения.


## 2. Реализация Megatron-LM в PyTorch

Чтобы использовать Megatron-LM в PyTorch, можно выполнить следующие шаги:

1. **Установить зависимости**: убедиться, что в среде установлены PyTorch, CUDA и NCCL, а также NVIDIA APEX. Для упрощения настройки окружения можно использовать NGC PyTorch container от NVIDIA.

2. **Клонировать репозиторий Megatron-LM**:
   ```bash
   git clone https://github.com/NVIDIA/Megatron-LM.git
   ```

3. **Установить Megatron-LM**: в директории клонированного репозитория установить зависимости Megatron-LM.
   ```bash
   cd Megatron-LM
   pip install -r requirements.txt
   ```

4. **Подготовить данные**: Megatron-LM поддерживает разные наборы данных, например Wikipedia, OpenWebText и другие. Нужно скачать набор данных и выполнить предварительную обработку, преобразовав его в формат, читаемый моделью.

5. **Обучить модель**: Megatron-LM предоставляет обучающие скрипты. Можно изменить конфигурационные файлы, чтобы задать размер модели, число эпох обучения, learning rate и другие параметры, а затем запустить процесс обучения.

6. **Оценка модели и инференс**: после завершения обучения можно использовать оценочные скрипты Megatron-LM для оценки производительности модели, а также выполнять генерацию текста или другие задачи инференса.


## 3. Реализация Megatron-LM в Accelerate:

При использовании Megatron-LM можно упростить процесс распределенного обучения с помощью библиотеки Accelerate от Hugging Face. Accelerate предоставляет простой интерфейс для интеграции DeepSpeed и Megatron-LM, облегчая распределенное обучение в PyTorch.

Ниже приведены базовые шаги для распределенного обучения с Megatron-LM и Accelerate:

1. **Установить Megatron-LM и Accelerate**:
   - Установить Megatron-LM и Accelerate через `pip`:
     ```bash
     pip install megatron-lm accelerate
     ```

2. **Подготовить окружение**:
   - Убедиться, что установлены последние версии PyTorch, CUDA, NCCL, а также дистрибутив NVIDIA APEX.
   - Можно использовать NVIDIA PyTorch container, предоставляемый NGC: он содержит все необходимые установки.

3. **Настроить Megatron-LM**:
   - Создать конфигурационный файл Megatron-LM, определяющий параметры модели, параметры обучения и т. д.
   - Настроить пути к данным, путь сохранения модели и другие параметры.

4. **Написать обучающий скрипт**:
   - Импортировать необходимые библиотеки и определить модель, загрузчик данных и оптимизатор.
   - Использовать объект `Accelerator` из Accelerate для подготовки модели, оптимизатора и загрузчика данных.

5. **Запустить обучение**:
   - Использовать команду `launch` из Accelerate для запуска распределенного обучения.
   - Указать конфигурационный файл, обучающий скрипт и необходимые параметры.

6. **Мониторинг и настройка**:
   - Во время обучения использовать инструменты Megatron-LM для мониторинга производительности и тюнинга.

7. **Сохранять и загружать checkpoints**:
   - Использовать механизм checkpoints Megatron-LM для сохранения и загрузки модели.

На момент завершения этой статьи (2024/10/14) поддержка Megatron-LM в Accelerate в основном сосредоточена на DP; в Accelerate пока нет PP и TP.


Ниже показана поддержка стратегий параллелизма разными фреймворками (по состоянию на 2024/10/14):

| Фреймворк | DP| PP |TP|3D-параллелизм|
| :--- |:----:| :----: |:---: |:---: |
| PyTorch(FSDP)|да|нет| нет|нет|
| DeepSpeed |да| да|да |да|
| Megatron-LM|да|да|да|да|
| Accelerate |да|нет|нет|нет|

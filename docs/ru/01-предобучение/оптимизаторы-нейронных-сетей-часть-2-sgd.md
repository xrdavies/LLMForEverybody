## 1. SGD
Стохастический градиентный спуск (Stochastic Gradient Descent, SGD) - это итерационный метод оптимизации дифференцируемой целевой функции, стохастическое приближение градиентного спуска. Истоки SGD восходят к 1951 году, когда Herbert Robbins и Sutton Monro в своей работе "A Stochastic Approximation Method" [1] впервые описали метод стохастической аппроксимации. Его можно считать предшественником SGD. Затем J. Kiefer и J. Wolfowitz в 1952 году опубликовали статью "Stochastic Estimation of the Maximum of a Regression Function" [2], которая ближе к современному пониманию SGD в машинном обучении.

Формула обновления для стохастического градиентного спуска (SGD) - это вариант метода градиентного спуска. Она используется для оптимизации целевой функции, особенно при работе с крупными наборами данных. На каждой итерации SGD использует только один пример или небольшой mini-batch примеров для вычисления градиента, а затем обновляет параметры модели. Это снижает вычислительную стоимость каждой итерации и помогает алгоритму выходить из локальных минимумов.

![alt text](../../../01-第一章-预训练/assest/神经网络的优化器（二）SGD/00.png)

Формула обновления параметров в SGD выглядит так:

$$\theta_{t+1} = \theta_t - \eta_t \nabla J(\theta_t; x^{(i)}, y^{(i)})$$

где:
- $\theta_t$ обозначает параметры модели на временном шаге t (это могут быть веса и смещения);
- $\eta_t$ обозначает learning rate, который управляет размером шага при обновлении параметров;
- $\nabla J(\theta_t; x^{(i)}, y^{(i)})$ обозначает градиент функции потерь J по параметрам $\theta_t$ на примере $(x^{(i)}, y^{(i)})$;
- $\theta_{t+1}$ - обновленные параметры модели.

На практике learning rate $\eta_t$ может быть фиксированным значением, а может постепенно уменьшаться со временем (learning rate decay), чтобы обеспечить стабильность и сходимость алгоритма во время обучения.

## 2. Недостатки SGD: неконвексные задачи оптимизации

Неконвексная задача оптимизации означает, что целевая функция имеет несколько локальных минимумов. В такой ситуации SGD может попасть в локальный минимум, из которого трудно выбраться. Причина в том, что SGD на каждой итерации использует только один пример или небольшой mini-batch для вычисления градиента; из-за этого направление градиента может оказаться неточным и повлиять на направление обновления параметров.

**Седловая точка**

![alt text](../../../01-第一章-预训练/assest/神经网络的优化器（二）SGD/0.png)

В оптимизации и машинном обучении седловая точка (Saddle Point) - это критическая точка целевой функции, в которой производные в одних направлениях положительны (подъем), а в других отрицательны (спуск). Иначе говоря, седловая точка не является ни локальным минимумом, ни локальным максимумом, а представляет собой некоторую комбинацию локального минимума и локального максимума.

Если говорить точнее:

- Локальный минимум: во всех окрестностях этой точки значение функции больше либо равно значению функции в этой точке;
- Локальный максимум: во всех окрестностях этой точки значение функции меньше либо равно значению функции в этой точке;
- Седловая точка: в одних направлениях окрестности этой точки значение функции больше либо равно значению в точке, а в других направлениях окрестности - меньше либо равно.

В двумерном пространстве седловую точку можно представить как форму седла: если идти от одной стороны седла к вершине, а затем на другую сторону, сначала вы пройдете участок подъема (признак локального минимума), а затем участок спуска (признак локального максимума).

В машинном обучении, особенно в глубоком обучении, наличие седловых точек может создавать трудности для градиентных методов оптимизации, таких как градиентный спуск. Алгоритм может застопориться около седловой точки, потому что градиент в этой точке равен нулю, и алгоритм не может определить, в каком направлении двигаться дальше.

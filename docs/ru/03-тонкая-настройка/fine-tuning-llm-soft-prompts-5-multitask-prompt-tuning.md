## Технический разбор

![alt text](../../../03-第三章-微调/assest/大模型微调之Soft prompts（五）Multitask prompt tuning/0.png)

Multitask Prompt Tuning (MPT) обучает одну подсказку на данных разных типов задач, и эту подсказку можно совместно использовать для разных целевых задач. Другие существующие методы обучают отдельную soft prompt для каждой задачи, а затем такие подсказки нужно извлекать или агрегировать, чтобы адаптироваться к целевой задаче. MPT включает два этапа:

1. Source training — для каждой задачи ее soft prompt разлагается на векторы, специфичные для задачи. Эти task-specific векторы перемножаются, образуя другую матрицу W, а затем между W и общей матрицей подсказки P используется произведение Адамара, чтобы получить матрицу подсказки, специфичную для задачи. Task-specific подсказка дистиллируется в единую матрицу подсказки, общую для всех задач. Эта подсказка обучается через многозадачное обучение.

2. Target adaptation — чтобы адаптировать единую подсказку к целевой задаче, целевая подсказка инициализируется и представляется как произведение Адамара общей матрицы подсказки и task-specific низкоранговой матрицы подсказки, то есть как поэлементное скалярное произведение.

## Интуитивное объяснение

Представьте, что студенту нужно подготовиться сразу к нескольким разным экзаменам, например по математике, истории и естественным наукам. В MPT этот студент похож на предобученную языковую модель, а экзамены похожи на разные downstream-задачи.

1. Изучение общих знаний: в начале семестра преподаватель дает студенту набор общих знаний, полезных для всех предметов, например приемы решения задач, методы письма или аналитические рамки. Это похоже на то, как в MPT модель обучает общую подсказку, которая помогает ей понимать разные типы задач.

2. Повторение под конкретные предметы: затем для экзамена по каждому предмету преподаватель может дать студенту специальные рекомендации по подготовке или ключевые темы, подобранные с учетом особенностей этого предмета. В MPT это означает, что модель немного подстраивает общую подсказку под особенности каждой задачи, чтобы лучше к ней адаптироваться.

3. Применение на экзамене: во время экзамена студент использует уже изученные общие знания и акценты подготовки для конкретного предмета, чтобы отвечать на вопросы. В MPT, сталкиваясь с новой задачей, модель использует уже выученную общую подсказку и корректирует ее под требования задачи, чтобы получить правильный output.

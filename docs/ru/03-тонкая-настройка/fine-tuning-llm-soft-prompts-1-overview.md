LoRA — не единственный вариант: секреты fine-tuning больших моделей через Soft Prompts, часть 1: обзор

## 1. Prompting

Обучение больших предобученных языковых моделей требует очень много времени и вычислительных ресурсов. По мере роста масштаба моделей все больше внимания получают более эффективные методы обучения, например Prompting. Prompting адаптирует замороженную предобученную модель к конкретной downstream-задаче с помощью текстовой подсказки, которая описывает задачу или показывает примеры ее выполнения. Благодаря Prompting можно не обучать отдельную модель полностью для каждой downstream-задачи, а использовать одну и ту же замороженную предобученную модель. Это гораздо проще: один и тот же модельный backbone можно применять к разным задачам, а обучение и хранение небольшого набора параметров prompt значительно эффективнее, чем обучение всех параметров модели.

## 2. Soft Prompts

Soft Prompts (мягкие подсказки) противопоставляются Hard Prompts (жестким подсказкам). Мягкие подсказки — это обучаемые непрерывные векторы, которые можно оптимизировать под конкретный датасет с помощью градиентных методов. Такой подход не требует ручного проектирования, может автоматически подстраиваться под разные задачи, вычислительно эффективен и поддерживает multitask learning. Однако мягкие подсказки нечитаемы, поэтому невозможно напрямую объяснить, почему были выбраны именно эти векторы.

Принцип работы мягких подсказок состоит в том, что на входном слое модели добавляется обучаемый проекционный слой, который отображает исходный вход в семантическое пространство, представленное prompt-информацией. Параметры проекционного слоя обучаются на тренировочных данных, благодаря чему prompt-информация лучше адаптируется к требованиям задачи.

## 3. Основные методы

### 3.1. Prompt Tuning

Ключевая идея Prompt Tuning [1](#refer-anchor-1) состоит в том, что prompt tokens имеют собственные параметры, которые можно обновлять независимо. Это значит, что параметры предобученной модели можно оставить неизменными и обновлять только градиенты embedding-векторов prompt tokens. Получаемый результат сопоставим с традиционным обучением всей модели, а по мере роста размера модели качество Prompt Tuning также повышается.

### 3.2 Prefix-Tuning

Prefix-Tuning [2](#refer-anchor-2) — вариант Prompt Tuning, который добавляет обучаемые prompt-векторы в позицию префикса входа модели. Преимущество этого метода в том, что для разных задач можно задавать разные подсказки, не меняя структуру модели.

Главное различие между Prefix-Tuning и Prompt Tuning заключается в том, что prefix-параметры в Prefix-Tuning вставляются во все слои модели, тогда как Prompt Tuning добавляет prompt-параметры только на embedding-слой модели.

![alt text](../../../03-第三章-微调/assest/大模型微调之Soft prompts（一）概述/1.png)

### 3.3 P-Tuning

P-tuning [3](#refer-anchor-3) в основном разработан для задач natural language understanding (NLU) и представляет собой еще один вариант Soft Prompts. P-tuning добавляет обучаемый embedding-тензор, который можно оптимизировать для поиска более удачной подсказки, а также использует prompt encoder (двунаправленную сеть long short-term memory, или LSTM) для оптимизации prompt-параметров.

Особенность P-tuning в том, что он позволяет моделям с Decoder-архитектурой адаптироваться к задачам Encoder-архитектуры, например к NLU-задачам.

![alt text](../../../03-第三章-微调/assest/大模型微调之Soft prompts（一）概述/2.png)

### 3.4 Multitask Prompt Tuning

Multitask Prompt Tuning (MPT) [4](#refer-anchor-4) — это метод, который обучает по данным одну подсказку, пригодную для нескольких типов задач; затем ее можно совместно использовать и адаптировать к разным целевым задачам. В отличие от него другие существующие методы обучают отдельный soft prompt для каждой задачи, а затем эти prompts нужно извлекать или агрегировать, чтобы адаптироваться к целевой задаче.

Проще говоря: MPT сначала обучает универсальную подсказку, а затем настраивает ее под конкретную задачу.

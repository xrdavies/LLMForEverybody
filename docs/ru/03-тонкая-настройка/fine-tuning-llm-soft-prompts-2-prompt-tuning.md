При fine-tuning не стоит знать только LoRA: Soft Prompts для больших моделей, часть 2: Prompt Tuning

В 2021 году понятие больших языковых моделей еще не было полностью устоявшимся, и понимание этой области находилось на стадии исследования. Среди множества исследовательских направлений Causal Language Model (модель только с декодером) была лишь одним из них. Тогда появление GPT-3 привлекло широкое внимание. Сразу после этого, в апреле того же года, Google представила метод Prompt Tuning, изначально протестированный на модели T5, которая использует encoder-decoder-архитектуру. В дальнейшем Prompt Tuning также показал свою эффективность на других downstream-задачах.


## Технический разбор

![alt text](../../../03-第三章-微调/assest/大模型微调之Soft prompts（二）Prompt-tuning/0.png)

Prompt Tuning — эффективный метод fine-tuning, который адаптирует модель к downstream-задачам, добавляя к входу модели специальные текстовые подсказки (prompt), без полного обновления параметров предобученной модели. Суть метода в том, что он настраивает поведение модели через оптимизацию параметров входной подсказки: модель лучше адаптируется к новой задаче, а основные параметры предобученной модели остаются неизменными.

Ключевое преимущество Prompt Tuning — эффективность по числу параметров. По сравнению с традиционными методами fine-tuning, Prompt Tuning требует обновления лишь небольшой части параметров, то есть параметров, связанных с подсказкой. Это заметно снижает потребность в вычислительных ресурсах и время обучения. Кроме того, Prompt Tuning может улучшать способность модели к обобщению, поскольку позволяет ей адаптироваться к новым задачам без большого объема размеченных данных.

Процесс Prompt Tuning обычно включает следующие шаги:

1. Выбрать предобученную модель в качестве основы;
2. Спроектировать или выбрать prompt-шаблон для конкретной задачи;
3. Объединить prompt-шаблон с входными данными, сформировав новую входную последовательность;
4. Провести обучение на предобученной модели, обновляя только параметры prompt-шаблона;
5. Оценить качество модели на тестовом датасете.


## Интуитивное объяснение

Ключевая идея Prompt Tuning состоит в том, что prompt tokens имеют собственные параметры, которые можно обновлять независимо. Это значит, что параметры предобученной модели можно оставить неизменными и обновлять только градиенты embedding-векторов prompt tokens. Получаемый результат сопоставим с традиционным обучением всей модели, а по мере роста размера модели качество Prompt Tuning также повышается.

## 1. Что такое тонкая настройка больших моделей

Тонкая настройка больших моделей (Fine-tuning) обычно означает дальнейшее обучение уже предобученной большой языковой модели (Large Language Models, LLMs) на специальном датасете, чтобы адаптировать модель к конкретной задаче или предметной области. Этот процесс позволяет модели изучить знания из нужной области и улучшить качество на конкретных NLP-задачах, например анализе тональности, распознавании сущностей, классификации текста, генерации диалогов и т. д.

1. **Предобученная модель**: до тонкой настройки большая модель обычно уже прошла масштабное unsupervised pretraining. Благодаря этому она усваивает базовые статистические свойства языка и знания, а также умеет предсказывать следующее слово.

2. **Датасет для конкретной задачи**: при тонкой настройке модель обучают на размеченных данных, связанных с конкретной задачей. Эти данные дают модели информацию из той области, которую ей нужно освоить.

3. **Настройка весов**: в процессе fine-tuning веса модели корректируются по данным конкретной задачи. Это может быть обновление всех параметров (Full Fine-tuning) или параметрически эффективная тонкая настройка (Parameter-Efficient Fine-Tuning, PEFT), при которой обновляется только часть параметров модели.

## 2. PEFT (Parameter-Efficient Fine-Tuning)

По сравнению с традиционными методами fine-tuning, PEFT эффективно снижает требования к вычислениям и памяти: он тонко настраивает лишь небольшую часть параметров модели, замораживая большую часть предобученной сети. Такая стратегия смягчает катастрофическое забывание у больших языковых моделей и заметно сокращает вычислительные затраты и расходы на хранение.

Основные методы PEFT см. по ссылкам на Adapters и Soft prompts.

## 3. Обзор фреймворков

1. **huggingface/peft**[1](#refer-anchor-1): базовый open source-инструмент от Hugging Face для параметрически эффективной тонкой настройки (Parameter-Efficient Fine-Tuning);

![alt text](../../../03-第三章-微调/assest/大模型微调框架（一）综述/0.png)

2. **modelscope/ms-swift**[2](#refer-anchor-2): легковесный open source-фреймворк от ModelScope для fine-tuning, в основном ориентированный на китайские большие модели и поддерживающий разные методы тонкой настройки. Fine-tuning можно запускать через скрипты или одной командой из кодовой среды; фреймворк поставляется с датасетами для тонкой настройки и валидации, поэтому можно быстро выполнить fine-tuning и проверку модели;

![alt text](../../../03-第三章-微调/assest/大模型微调框架（一）综述/2.png)

3. **hiyouga/LLaMA-Factory**[3](#refer-anchor-3): full-stack-инструмент для тонкой настройки, который поддерживает множество моделей и разные популярные методы fine-tuning. Он позволяет запускать тонкую настройку скриптами, работать через Web-интерфейс и содержит базовые тренировочные датасеты. Помимо fine-tuning, также поддерживает инкрементальное предобучение и полную тонкую настройку;

![alt text](../../../03-第三章-微调/assest/大模型微调框架（一）综述/3.png)

4. **NVIDIA/Megatron-LM**[4](#refer-anchor-4): фреймворк для обучения больших моделей, разработанный NVIDIA. Он поддерживает масштабное pretraining и fine-tuning и подходит для обучения и тонкой настройки больших моделей, которым нужны очень высокая производительность и масштаб.

## 4. Библиотека PEFT

PEFT (параметрически эффективная тонкая настройка) — это библиотека для эффективной адаптации больших предобученных моделей к разным downstream-приложениям без тонкой настройки всех параметров модели, поскольку это слишком дорого. Методы PEFT тонко настраивают лишь небольшое число (дополнительных) параметров модели, при этом достигая качества, сопоставимого с полной тонкой настройкой. Благодаря этому обучать и хранить большие языковые модели (LLM) на потребительском оборудовании становится проще.

![alt text](../../../03-第三章-微调/assest/大模型微调框架（一）综述/1.png)

PEFT интегрируется с библиотеками Transformers, Diffusers и Accelerate, предоставляя более быстрый и простой способ загружать, обучать и использовать большие модели для инференса.

Использование библиотеки PEFT можно свести к следующим шагам:

### 1. Установка библиотеки PEFT

Библиотеку PEFT можно установить через PyPI следующей командой:
```bash
pip install peft
```
Либо, если нужно установить из исходного кода, чтобы получить самые свежие функции, можно использовать команду:
```bash
pip install git+https://github.com/huggingface/peft
```
Пользователи, которые хотят вносить вклад в код или смотреть актуальные результаты, могут клонировать репозиторий с GitHub и установить editable-версию:
```bash
git clone https://github.com/huggingface/peft
cd peft
pip install -e .
```


### 2. Настройка метода PEFT
Каждый метод PEFT определяется классом `PeftConfig`, который хранит все важные параметры для создания `PeftModel`. На примере LoRA нужно указать тип задачи, режим инференса, размерность низкоранговых матриц и другие параметры:
```python
from peft import LoraConfig, TaskType
peft_config = LoraConfig(task_type=TaskType.SEQ_2_SEQ_LM, inference_mode=False, r=8, lora_alpha=32, lora_dropout=0.1)
```


### 3. Загрузка предобученной модели и применение PEFT
Загрузите базовую модель для тонкой настройки и используйте функцию `get_peft_model()`, чтобы обернуть базовую модель вместе с `peft_config` и создать `PeftModel`:
```python
from transformers import AutoModelForSeq2SeqLM
from peft import get_peft_model
model = AutoModelForSeq2SeqLM.from_pretrained("bigscience/mt0-large")
model = get_peft_model(model, peft_config)
```


### 4. Обучение модели
Теперь модель можно обучать с помощью `Trainer` из Transformers, Accelerate или любого пользовательского training loop на PyTorch. Например, обучение через класс `Trainer`:
```python
from transformers import TrainingArguments, Trainer
training_args = TrainingArguments(
    output_dir="your-name/bigscience/mt0-large-lora",
    learning_rate=1e-3,
    per_device_train_batch_size=32,
    num_train_epochs=2,
    weight_decay=0.01,
)
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_datasets["train"],
    eval_dataset=tokenized_datasets["test"],
    tokenizer=tokenizer,
    data_collator=data_collator,
    compute_metrics=compute_metrics,
)
trainer.train()
```


### 5. Сохранение и загрузка модели
После завершения обучения модель можно сохранить в каталог с помощью функции `save_pretrained` или отправить на Hugging Face Hub с помощью функции `push_to_hub`:
```python
model.save_pretrained("output_dir")
from huggingface_hub import notebook_login
notebook_login()
model.push_to_hub("your-name/bigscience/mt0-large-lora")
```


### 6. Инференс
Используйте класс `AutoPeftModel` и метод `from_pretrained`, чтобы легко загрузить любую inference-модель, обученную с помощью PEFT:
```python
from peft import AutoPeftModelForCausalLM
from transformers import AutoTokenizer
model = AutoPeftModelForCausalLM.from_pretrained("ybelkada/opt-350m-lora")
tokenizer = AutoTokenizer.from_pretrained("facebook/opt-350m")
inputs = tokenizer("Preheat the oven to 350 degrees and place the cookie dough", return_tensors="pt")
outputs = model.generate(input_ids=inputs["input_ids"], max_new_tokens=50)
print(tokenizer.batch_decode(outputs.detach().cpu().numpy(), skip_special_tokens=True)[0])
```

## 1. Web-UI

LLaMA Factory предоставляет удобный визуальный интерфейс (WebUI), который позволяет пользователям тонко настраивать большие языковые модели без написания кода.

**Обзор интерфейса**:
   - WebUI в LLaMA Factory в основном состоит из четырех экранов: обучение, оценка и предсказание, диалог, экспорт.

**Экран обучения**:
   - На экране обучения пользователь может указать имя и путь модели, этап обучения, метод fine-tuning, тренировочный датасет, learning rate, число эпох и другие параметры обучения, а затем запустить обучение.
   - Поддерживается возобновление с checkpoint; checkpoint адаптера сохраняется в каталоге `output_dir`.

**Экран оценки, предсказания и диалога**:
   - После завершения обучения модели пользователь может на экране оценки и предсказания указать пути к модели и адаптеру и выполнить оценку на заданном датасете.
   - Пользователь также может общаться с моделью через экран диалога и наблюдать за качеством модели.

**Экран экспорта**:
   - Если пользователь доволен качеством модели и хочет экспортировать ее, он может на экране экспорта указать модель, адаптер, размер шардов, уровень экспортной квантизации и calibration dataset, устройство экспорта, каталог экспорта и другие параметры, а затем нажать кнопку экспорта.

**Шаги использования**:
   - Пользователь может открыть в браузере `http://localhost:7860`, перейти в Web-интерфейс, выбрать модель, настроить параметры и отслеживать ход тренировочной задачи.
   - Поддерживается переключение языка на китайский, что удобно для пользователей в Китае.

**Поддерживаемые модели и датасеты**:
   - LLaMA Factory поддерживает разные предобученные модели, например LLaMA, BLOOM, Mistral и другие, а также может автоматически скачивать и кэшировать ресурсы моделей и датасетов из ModelScope.

**Методы fine-tuning**:
   - Доступны разные методы тонкой настройки, включая full-parameter fine-tuning, freeze fine-tuning, LoRA fine-tuning и другие.

**Настройка параметров**:
   - WebUI позволяет пользователю оптимизировать модель с помощью простого перетаскивания и настройки параметров, поэтому даже люди без опыта программирования могут легко участвовать в оптимизации модели.

## 2. За пределами LoRA

LLaMA-Factory предоставляет набор продвинутых функций для сложной тонкой настройки и развертывания больших моделей:

**Совместимость со множеством моделей**: LLaMA-Factory поддерживает разные большие языковые модели, например LLaMA, BLOOM, Mistral, Baichuan, Qwen, ChatGLM и другие;

**Интеграция алгоритмов обучения**: фреймворк интегрирует разные алгоритмы обучения, включая инкрементальное предобучение, supervised fine-tuning на инструкциях, обучение reward model, PPO training, DPO training, KTO training, ORPO training и другие;

**Точность вычислений и алгоритмы оптимизации**: доступны разные варианты вычислительной точности, например 32-битная full-parameter fine-tuning, 16-битная freeze fine-tuning, 16-битная LoRA fine-tuning, а также 2/4/8-битная QLoRA fine-tuning на основе AQLM/AWQ/GPTQ/LLM.int8. Также поддерживаются продвинутые алгоритмы GaLore, DoRA, LongLoRA, LLaMA Pro, LoRA+, LoftQ и Agent fine-tuning;

**Поддержка inference engines**: поддерживаются inference engines Transformers и vLLM, что дает пользователям гибкие варианты инференса;

**Интеграция experiment dashboards**: интеграция с LlamaBoard, TensorBoard, Wandb, MLflow и другими инструментами experiment dashboards упрощает мониторинг и анализ процесса обучения;

**Функция API Server**: LLaMA-Factory поддерживает запуск API Server, позволяя пользователям вызывать модель через API, что упрощает интеграцию и применение модели;

**Benchmark для оценки больших моделей**: предоставляет популярные benchmark-инструменты для оценки больших моделей, которые помогают пользователю измерять качество модели;

**Установка через Docker и адаптация к Huawei NPU**: поддерживается установка через Docker и использование на Huawei NPU, что повышает переносимость фреймворка и совместимость с аппаратным обеспечением;

**Технологии квантизации**: поддерживаются PTQ, QAT, AQLM, OFTQ и другие технологии квантизации, оптимизирующие эффективность и качество развертывания модели;

Эти продвинутые функции позволяют LLaMA-Factory поддерживать не только базовую тонкую настройку моделей, но и более сложные исследовательские и прикладные задачи, предоставляя пользователям мощный и гибкий набор инструментов.

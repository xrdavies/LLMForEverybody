LoRA — не единственный вариант: секреты fine-tuning больших моделей через Soft Prompts, часть 4: P-Tuning

P-Tuning — это метод Soft Prompts, разработанный для задач NLU. P-tuning добавляет обучаемый embedding-тензор, который можно оптимизировать для поиска более удачной подсказки, а также использует prompt encoder, например BiLSTM+MLP, для оптимизации параметров подсказки.

## Технический разбор

![alt text](../../../03-第三章-微调/assest/大模型微调之Soft prompts（四）P-tuning/0.png)

У P-tuning есть две версии: P-tuning v1 (2021 год) и P-tuning v2 (2023 год). P-tuning v1 оптимизирует параметры prompt с помощью prompt encoder, например BiLSTM+MLP, но этот подход плохо работает на некоторых сложных задачах понимания естественного языка (NLU) и показывает хорошие результаты только при достаточно большом размере модели.

Чтобы решить эти проблемы, был предложен P-tuning v2. P-tuning v2 улучшает v1: он добавляет обучаемые непрерывные подсказки не только на входном слое, но и в каждый слой модели. Благодаря этому метод лучше адаптируется к разным задачам, включая сложные задачи NLU. P-tuning v2 оптимизирует параметры подсказки с помощью стратегии многозадачного обучения и глубокого prompt encoder, поэтому на моделях разных размеров он может достигать качества, сопоставимого с полной тонкой настройкой всех параметров.

## Интуитивное объяснение

Главная особенность P-Tuning в том, что за счет введения BiLSTM/MLP модель может лучше справляться с задачами NLU. Преимущество этого подхода заключается в том, что он позволяет без изменения структуры модели придать моделям decoder-архитектуры свойства encoder-архитектуры.
